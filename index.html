DUNG EONS & AI — ПОЛНЫЙ ДИЗАЙН-ДОКУМЕНТ Ролевая игра с ИИ-мастером на бесплатных API --- ЧАСТЬ 1. ТЕХНИЧЕСКАЯ ОСНОВА 1.1 Бесплатные API и архитектура прокси-сервера «Гирлянда» Игра Dung Eons & AI работает на связке из четырёх полностью бесплатных API, каждый из которых не требует привязки банковской карты для начала использования. Это фундаментальное требование проекта: игра должна запускаться и работать без финансовых вложений со стороны разработчиков. Прокси-сервер развёрнут на платформе Cloudflare Workers и автоматически переключается между провайдерами при исчерпании дневных лимитов запросов или возникновении технических ошибок на стороне провайдера. Архитектура получила название «Гирлянда» потому что провайдеры соединены в последовательную цепочку подобно лампочкам в ёлочной гирлянде: когда одна перегорает (провайдер исчерпывает лимит или начинает выдавать ошибки), загорается следующая, и игровой процесс не прерывается. Для игрока переключение абсолютно незаметно — время ответа может измениться на 100-200 миллисекунд, но это не вызывает видимых задержек. Четыре провайдера были отобраны после тщательного анализа рынка бесплатных API для больших языковых моделей. Ключевой критерий отбора — отсутствие требования привязывать банковскую карту для активации бесплатного тарифа. Многие популярные сервисы требуют верификации платёжного метода даже для использования бесплатных кредитов, что делает их непригодными для проекта. Используемые API по порядку приоритета: Первое место — Groq с моделью Llama 3.3 70B. Groq предоставляет самый щедрый бесплатный лимит среди всех рассмотренных провайдеров — 14 400 запросов в день. Модель Llama 3.3 70B разработана компанией Meta, имеет 70 миллиардов параметров и оптимизирована для быстрого инференса на специализированном оборудовании Groq LPU (Language Processing Unit). Скорость ответа составляет 200-400 миллисекунд, что является одним из лучших показателей на рынке. Модель отлично справляется с задачами творческого письма (описание сцен, генерация диалогов NPC), точного следования инструкциям (критически важно для соблюдения DDS-протокола), и ролевого отыгрыша (важно для создания убедительных персонажей). Второе место — Cerebras с моделью Qwen-3-235B. Cerebras предоставляет примерно такой же лимит — около 14 400 запросов в день. Qwen-3-235B это модель с 235 миллиардами параметров, разработанная компанией Alibaba специально для многоязычных задач, включая русский язык. Это делает её особенно ценной для игры где многие игроки будут общаться на русском. Ключевое преимущество Cerebras — скорость инференса до 1000 токенов в секунду на специализированном оборудовании Wafer-Scale Engine. Это делает модель идеальной для интенсивных боевых сцен где ИИ нужно сгенерировать множество DDS-команд за минимальное время. Третье место — Google AI Studio с моделью Gemini 2.5 Flash. Google предоставляет 1 500 бесплатных запросов в день — заметно меньше чем у лидеров, но модель обладает уникальным преимуществом: отличное понимание сверхдлинных контекстов. Контекст включает полную карту блоков локации, информацию о всех игроках, NPC, объектах, флагах, историю — это может занимать десятки тысяч токенов. Gemini 2.5 Flash специально оптимизирована для работы с такими объёмами данных и реже теряет важные детали из середины длинного промпта. Четвёртое место — Cohere с моделью Command R+. Cohere предоставляет всего 1 000 запросов в месяц, что несопоставимо с дневными лимитами лидеров. Эта модель используется исключительно как аварийный резерв: когда все три основных провайдера одновременно исчерпали дневные лимиты или недоступны по техническим причинам. В обычных условиях до Cohere очередь не доходит. Трёхзвенная архитектура: Игровой клиент (приложение на устройстве игрока) → Игровой сервер (обработка игровой логики и бросков кубиков) → Прокси-сервер (Cloudflare Workers) → API провайдеров (Groq / Cerebras / Google / Cohere) Почему нельзя отправлять запросы напрямую с игрового сервера к API провайдерам минуя прокси: Причина первая — безопасность хранения API-ключей. Каждый провайдер выдаёт уникальный секретный ключ который аутентифицирует нас при каждом запросе. Этот ключ по сути пароль от аккаунта. Если ключ попадёт в руки злоумышленников, они смогут использовать наш аккаунт для генерации любого контента включая запрещённый, что приведёт к немедленной блокировке. Cloudflare Workers решают эту проблему тем что ключи хранятся в переменных окружения платформы (environment variables). Эти переменные шифруются на стороне Cloudflare, доступ к ним имеет только владелец аккаунта Cloudflare, и они никогда не передаются в клиентский код. Даже если злоумышленник получит полный доступ к исходному коду воркера, он не сможет прочитать значения переменных окружения. Причина вторая — надёжность переключения между провайдерами. Логика переключения требует хранения состояния: сколько запросов уже отправлено каждому провайдеру за сегодня, у какого провайдера какие ошибки возникали, кого нужно временно заблокировать. Если бы эта логика жила на игровом сервере, при любом перезапуске сервера все счётчики сбрасывались бы в ноль, и система забывала бы что Groq уже исчерпан. Cloudflare Durable Objects решают эту проблему — они сохраняют состояние между запросами и даже между перезапусками воркера. Причина третья — оптимизация трафика. Ответы от API провайдеров приходят в формате JSON и могут быть довольно объёмными (несколько килобайт на каждый ответ). Прокси сжимает эти ответы алгоритмом gzip перед отправкой на игровой сервер, уменьшая объём передаваемых данных в 3-5 раз. Также прокси может вырезать из ответа техническую информацию которая не нужна игровому серверу. Причина четвёртая — географическая близость. Cloudflare имеет точки присутствия в более чем 300 городах по всему миру. Когда игровой сервер отправляет запрос на наш прокси, этот запрос автоматически направляется к ближайшей точке Cloudflare (обычно в том же городе или регионе). А Cloudflare в свою очередь направляет запрос к ближайшей точке API провайдера. За счёт этого суммарная задержка снижается до 200-400 миллисекунд, тогда как прямой запрос мог бы идти через множество промежуточных узлов с непредсказуемой задержкой до 800-1200 миллисекунд. Семь модулей прокси-сервера: Модуль первый — маршрутизация запросов. Когда игровой сервер отправляет POST-запрос на адрес нашего прокси, запрос содержит в теле JSON с двумя полями: "context" (полный контекст игры согласно структуре описанной ниже) и "action" (текст который игрок ввёл в чат). Прокси получает этот запрос, извлекает из заголовков информацию о том какая кампания и какой игрок, определяет по внутреннему состоянию Durable Object какой провайдер сейчас активен, достаёт API-ключ этого провайдера из переменных окружения, добавляет ключ в HTTP-заголовок Authorization согласно формату который требует данный провайдер, и отправляет запрос на API-эндпоинт провайдера. Получив ответ, прокси извлекает из него сгенерированный текст и DDS-команды, упаковывает их в JSON и отправляет обратно на игровой сервер. Модуль второй — автоматическое переключение провайдеров. Durable Object хранит для каждого провайдера счётчик использованных запросов за текущие сутки. При каждом запросе счётчик активного провайдера увеличивается на единицу. Если счётчик достигает дневного лимита (14 400 для Groq, примерно 14 400 для Cerebras, 1 500 для Google, 1 000 для Cohere), флаг активного провайдера переключается на следующего в цепочке. Сброс счётчиков происходит в полночь по всемирному координированному времени (UTC), и в этот момент цепочка сбрасывается до начала — активным снова становится Groq. Модуль третий — кэширование ответов. Если два игрока в одной партии одновременно отправляют идентичные запросы (что случается при сетевых лагах когда клиент думает что запрос не ушёл и отправляет повторно), прокси сохраняет ответ от первого запроса в кэш на 2 секунды и для второго запроса возвращает кэшированный ответ вместо повторной отправки в API. Это экономит лимиты запросов. Ключом для кэша служит хеш от полного содержимого запроса. Модуль четвёртый — сжатие DDS-ответов. Ответы от ИИ содержат множество DDS-команд которые могут занимать несколько килобайт. Прокси применяет алгоритм сжатия gzip к телу ответа перед отправкой на игровой сервер. Это уменьшает объём передаваемых данных в 3-5 раз и ускоряет загрузку ответа на клиенте. Модуль пятый — мониторинг использования. Прокси записывает в логи каждый запрос: точное время, идентификатор кампании, имя провайдера, длительность обработки, количество токенов во входе и выходе, статус ответа. Если процент ошибок у какого-то провайдера превышает 5% за последние 10 минут, прокси отправляет алерт разработчику через вебхук в мессенджер. Также отслеживается скорость расхода дневных лимитов: если к полудню UTC израсходовано более 80% лимита, это сигнал что нагрузка растёт и скоро может потребоваться добавление новых провайдеров. Модуль шестой — Rate limiting. Помимо дневных лимитов, провайдеры устанавливают лимиты на количество запросов в минуту (обычно от 20 до 100). Прокси отслеживает эти лимиты с помощью алгоритма скользящего окна: хранит временные метки последних N запросов и если количество запросов за последнюю минуту приближается к лимиту, начинает ставить входящие запросы в очередь с задержкой в 1-2 секунды чтобы не превысить лимит. Модуль седьмой — географическая близость. Cloudflare Workers автоматически запускают код воркера в ближайшей к отправителю точке присутствия. За счёт этого запрос от игрового сервера в Европе будет обработан в европейской точке Cloudflare и направлен к европейскому же эндпоинту API провайдера, а запрос от сервера в Азии пойдёт через азиатскую точку. Это минимизирует сетевую задержку для игроков из любого региона. Обработка ошибок: Ошибка 429 (Too Many Requests, Rate Limit). Провайдер сообщает что мы превысили лимит запросов в минуту. Прокси не переключается сразу на следующего провайдера, потому что эта ошибка временная и обычно длится всего несколько секунд. Вместо этого прокси ждёт ровно 1 секунду и пробует отправить тот же запрос повторно. Если три попытки подряд завершаются ошибкой 429, значит лимит исчерпан надолго и прокси переключается на следующего провайдера. При этом прокси запоминает что этот провайдер находится под ограничением и не пытается использовать его в течение следующих 5 минут. Ошибка 500 (Internal Server Error). Означает что на стороне провайдера произошёл сбой — перегрузка серверов, ошибка в программном обеспечении, или проблемы с сетью внутри дата-центра. Такие ошибки обычно не проходят за секунду, поэтому прокси мгновенно переключается на следующего провайдера без повторных попыток. Таймаут (отсутствие ответа в течение 30 секунд). Также приводит к мгновенному переключению. Таймаут в 30 секунд выбран с большим запасом: обычно API отвечают за 1-5 секунд, и 30 секунд достаточно чтобы покрыть даже самые медленные ответы на сложные запросы с длинным контекстом. Если все четыре провайдера вернули ошибку или недоступны — активируется экстренный fallback-режим. Fallback-режим — автономная работа без ИИ. Когда ни одно из API недоступно (ситуация экстраординарная — например глобальный сбой интернета или скоординированная атака на всех провайдеров), в коде прокси хранятся заранее заготовленные шаблонные DDS-ответы для базовых игровых ситуаций. Полный список того что может обработать автономный режим: · Атака попала ближним боем — сервер уже знает результат броска и урон (они посчитаны локально), поэтому персонаж наносит урон врагу, но без красивого описания от ИИ. Игрок видит сообщение «Попадание! Нанесено 8 урона.» · Атака попала дальним боем — аналогично, урон применяется без художественного описания. · Атака промахнулась — персонаж делает анимацию атаки но урон не наносится, сообщение «Промах!» · Движение вперёд, назад, в сторону — персонаж перемещается на указанное количество клеток. · Обыск локации — если в локации нет скрытых объектов, возвращается «Вы осматриваетесь но не находите ничего примечательного». Скрытые объекты не обнаруживаются. · Открытие сундука — сервер использует локальную таблицу добычи для генерации содержимого. · Диалог с NPC — NPC отвечает одной из трёх базовых реплик в зависимости от отношения: «NPC приветливо кивает» (дружелюбие), «NPC смотрит на вас с безразличием» (нейтрально), «NPC хмурится и отворачивается» (неприязнь). · Использование предмета — зелье лечения восстанавливает стандартное количество HP, яд наносит стандартный урон. · Отдых — персонаж восстанавливает HP и ячейки заклинаний по стандартным правилам. Если игрок пытается сделать что-то что выходит за рамки автономного режима (сложное заклинание требующее описания, переговоры с боссом, генерация новой локации), сервер показывает окно с сообщением о том что ресурсы ИИ исчерпаны, предлагает сохранить прогресс и выйти до сброса лимитов. Автономный режим позволяет продолжить базовые действия но не даёт продвигаться по сюжету. При первой же возможности прокси переподключается к живому API и игра возвращается в нормальный режим. Что кэшируется: · System-сообщения (промпты) — через хеш содержимого. Если промпт не изменился с прошлого запроса, токены за него не считаются. Это даёт огромную экономию потому что промпт занимает десятки тысяч токенов. · Ответы на полностью идентичные запросы с временем жизни 2 секунды. Что не кэшируется никогда: · Любые запросы содержащие состояние мира или персонажа (HP, позиции, эффекты). · Боевые запросы потому что каждый бросок кубика уникален. · Запросы с разными user-сообщениями, даже если они отличаются на один символ. Две модели задействованы в игре: Основная модель обслуживает 95% всех запросов: описания локаций, диалоги с NPC, бои с простыми врагами, генерация стандартных комнат по шаблонам, базовые анимации, саммаризация истории партии. Эти запросы идут через того провайдера который сейчас активен в цепочке — неважно Groq это, Cerebras или Google. Для особо сложных сцен предусмотрен отдельный маршрут: битвы с боссами, создание уникальных локаций с нуля, нестандартные сюжетные повороты требующие креативности. Такие запросы прокси всегда пытается направить на Groq с Llama 3.3 70B которая показывает лучшие результаты в творческих задачах. Если Groq недоступен — запрос уходит на Cerebras с Qwen-3-235B как запасной вариант для сложных сцен. Выбор версии правил и промпта: При создании новой кампании ведущий или первый игрок (если ведущий не назначен) выбирает одну из трёх редакций правил Dungeons & Dragons. На выбор предлагаются: редакция 3.5 (самая глубокая и комплексная, с сотнями навыков прокачиваемых очками и детальными числовыми модификаторами), редакция 5.0 2014 года (сбалансированная популярная версия с системой преимущества/помехи), и редакция 5.5 2024 года (осовремененная версия где раса не определяет характеристики а все классы получают подкласс на 3 уровне). Сервер на основе этого выбора подставляет один из трёх соответствующих промптов в system-сообщение ИИ. Промпт включает полные правила выбранной редакции, DDS-протокол со всеми командами, бестиарий, артефакты, инструкции по генерации локаций и NPC, правила создания имён, правила модерации. Серверный промпт-контекст — полная структура данных отправляемая ИИ при каждом запросе. Сервер формирует JSON-объект описанной ниже структуры и отправляет его в поле "context" каждого запроса к ИИ через прокси. Прокси преобразует этот JSON в текстовое представление которое подставляется в секцию [КОНТЕКСТ] промпта ИИ. ИИ работает ТОЛЬКО с данными которые получает в этом конкретном запросе. ИИ не помнит что происходило в предыдущих запросах. Поэтому сервер обязан включать в контекст абсолютно всё что может понадобиться для принятия решений, описания сцен и создания сюжетных поворотов. Справа от полосы числа: «45/45» (текущие/максимальные). Если есть временные хиты — они отображаются голубой полосой над основной. 2. Имя персонажа. Крупным шрифтом. Под именем мелким серым — никнейм игрока в скобках. «Алан (AlanTheBrave)». 3. Раса и подраса. «Человек», «Эльф (лесной)», «Дварф (горный)», «Драконорождённый (красный)». 4. Класс и архетип. «Воин (Чемпион)», «Волшебник (Школа Эвокации)». Если архетип ещё не выбран — «(архетип на 3 уровне)». 5. Уровень с полоской опыта. «Ур: 5 [█████░░░░░] 6500/14000». Заполнение полоски опыта = текущий опыт / опыт до следующего уровня. 6. AC (Armor Class — Класс Брони). «AC: 18». Временные бонусы в скобках: «AC: 18 (+2 от щита веры)». Под ним мелко расшифровка: «(Кольчуга 16 + Щит 2 + Ловк 0)». 7. Инициатива. «Иниц: +3». Модификатор для броска порядка ходов в бою. 8. Скорость. «Скор: 30 футов». При изменениях: «Скор: 20 футов (базовая 30, -10 от тяжёлой брони)». 9. Все шесть базовых характеристик. Каждая с полным названием, числовым значением и модификатором в скобках. «СИЛА 18 (+4)», «ЛОВК 14 (+2)», «ТЕЛО 16 (+3)», «ИНТЕЛ 10 (+0)», «МУДР 12 (+1)», «ХАРИЗ 8 (-1)». Модификатор вычисляется по формуле: (значение - 10) / 2 с округлением вниз. 10. Бонус мастерства. «Бонус маст: +3». Растёт с уровнем: +2 на 1-4 уровнях, +3 на 5-8, +4 на 9-12, +5 на 13-16, +6 на 17-20. 11. Пассивная внимательность. «Пасс вним: 14». Формула: 10 + модификатор Мудрости + бонус мастерства (если есть владение навыком Внимательности). 12. Кости хитов. «Кости хитов: 5/5 d10». Осталось/максимум, тип кости зависит от класса. 13. Владения. Списки через запятую. «Владения оружием: Все виды оружия», «Владения бронёй: Тяжёлая броня, средняя, лёгкая, щиты», «Владения инструментами: Инструменты кузнеца». 14. Языки. «Языки: Общий, Дварфийский». 15. Ячейки заклинаний. ТОЛЬКО для заклинателей. «Яч. 1: 3/4», «Яч. 2: 2/2». Потраченные — серым, доступные — ярким. 16. Экипировка. «Правая: Дл. меч 1d8», «Левая: Щит +2 AC», «Броня: Кольчуга AC 16», «Арт. 1: Кольцо защиты +1 AC», «Арт. 2: пусто», «Арт. 3: пусто», «Золото: 450». 17. Активные эффекты. Баффы зелёным, дебаффы красным, нейтральные синим. С описанием и таймером. «• Благословение: +1d4 к атакам (5 мин 30 с)», «• Концентрация: Магическое оружие (активна)». 18. Кнопки. «⚙ НАСТРОЙКИ», «[Профиль]», «[Спос-ти]», «[Инвент]», «[Тактич.]». Три чата — столбиком снизу вверх в левом нижнем углу. Самый нижний — Чат №1, над ним Чат №2, над ним Чат №3. Порядок: 1 → 2 → 3. Ширина примерно 30% экрана, высота примерно 40%. Фон каждого чата — полупрозрачный тёмный с размытием. По умолчанию видны последние 2 сообщения. Кнопка [□] разворачивает чат на весь экран. Кнопка [↕] расширяет чат перекрывая соседние. Чат №1 — Игровой чат. Содержит: нарратив ИИ (белый цвет, префикс [ИИ]:), действия игроков (цвет игрока, префикс [Имя]:), результаты бросков (жёлтый, дублируются КРУПНЫМ ТЕКСТОМ в центре экрана на 3 секунды с анимацией), системные сообщения (серый), DDS-команды в человекочитаемом виде (сервер преобразует /Ba23X5YA в «Построен каменный блок на позиции 5,10»). Поле ввода внизу чата. Блокируется на время ожидания ИИ. Чат №2 — Вопросы к ИИ. ПОДКЛЮЧЕН к ИИ. Для технических вопросов о механике и правилах. Сообщения отправляются с пометкой [ВОПРОС ВНЕ ИГРЫ]. ИИ отвечает описательно но не выполняет игровых действий. Ответы видны ТОЛЬКО спросившему игроку. Примеры: «Сколько мне надо выбросить чтобы убить этого гоблина с моим копьём?», «Какой радиус у Огненного шара на 5 уровне?». Чат №3 — Общий чат. ПОЛНОСТЬЮ ИЗОЛИРОВАН от ИИ. Сообщения никогда не отправляются в API. Для свободного общения между игроками. НЕ ИМЕЕТ ФИЛЬТРОВ — игроки общаются как хотят. Голосовой чат. Встроенный голосовой чат по принципу рации (нажать кнопку — говорить). НЕ ИМЕЕТ ФИЛЬТРОВ. Громкость голоса влияет на дальность слышимости в игре. Стены блокируют звук на основе геометрии локации. Тактический стол. Инструмент для детального планирования. Не заменяет общение (игроки всегда могут говорить). Активируется кнопкой [Тактич.]. Сцена замирает, появляется стол с миниатюрой видимой локации. Игроки двигают фигурки, рисуют стрелки, используют несколько уровней высоты. Всё на столе — чистая визуализация, не влияет на игру. Кнопка «Готов» возвращает в игру. Нижняя панель — полоса персонажей. Горизонтальная панель во всю ширину экрана в самом низу. Показывает всех участников сцены: игроков, союзных NPC, врагов. Для каждого: иконка лица из блоков, имя, полоска HP с числами, уровень. В бою: ЗЕЛЁНАЯ рамка на текущем ходящем, ЖЁЛТАЯ на следующем. Круговой таймер хода. При клике на игрока — открывается полный профиль. При клике на врага — базовая информация (тип, состояние здоровья словами, видимые эффекты). Адаптация под устройства. На смартфонах: правая панель скрыта и выдвигается свайпом влево, чаты в нижней трети. На планшетах: зависит от ориентации. Поддерживается голосовой ввод текста через системную клавиатуру. --- 1.3 Распределение ответственности между сервером и ИИ Игроки всегда могут общаться друг с другом. Чат №3 и голосовой чат не имеют ограничений и всегда доступны. Игроки обсуждают стратегию, делятся впечатлениями, шутят — это часть социального опыта игры и не требует фильтрации. Фильтры применяются ТОЛЬКО к сообщениям которые отправляются ИИ — это содержимое Чата №1 (игровые действия) и Чата №2 (вопросы к ИИ). Сервер проверяет эти сообщения на наличие запрещённых категорий перед отправкой в API. Фильтруются: · Нецензурная лексика (мат) на русском и английском языках. · Расистские и дискриминационные выражения. · Откровенный сексуальный контент (18+). · Раскрытие личной информации (email, телефоны, адреса, паспортные данные). · Скам-ссылки и попытки фишинга. · Попытки эксплуатации уязвимостей сервера (SQL-инъекции, XSS, command injection). ВАЖНО: Призывы к насилию в реальном мире НЕ ФИЛЬТРУЮТСЯ. Игровые описания насилия («убиваю гоблина», «отрубаю голову орку») — это нормальный игровой процесс. ИИ имеет следующие ограничения в промпте: 1. Не отвечать на запросы не связанные с игрой. Ответ: «Я — ИИ-ведущий Dung Eons & AI. Я могу помочь только с игровыми вопросами.» 2. Не раскрывать что он ИИ. 3. Не давать метагейминговых советов. 4. Не предсказывать сюжет в явном виде. 5. Не выходить за рамки правил D&D. 6. Пресекать попытки игроков управлять миром: «Ты не ведущий, чтобы указывать.» Цепочка обработки действия из 7 шагов: Шаг 1. Игрок вводит текст действия в поле ввода Чата №1 и нажимает Enter. Шаг 2. Сервер формирует пакет: берёт [КОНТЕКСТ] в формате JSON, добавляет [ДЕЙСТВИЕ ИГРОКА] с пометкой [ДЕЙСТВИЕ] или [ВОПРОС], отправляет POST-запрос на прокси который перенаправляет его в активный API. Шаг 3. ИИ анализирует текст и возвращает DDS-команду /R[тип] с классификацией действия и DC если нужна проверка навыка. Шаг 4. Сервер получив классификацию делает все необходимые броски кубиков: бросает d20, прибавляет модификаторы характеристик и бонус мастерства, сравнивает с DC или AC, бросает кубы урона если атака попала. Шаг 5. Сервер отправляет ИИ структурированные результаты: [A] для атаки, [Урон] для урона, [Состояние цели] для изменения HP, [УСПЕХ]/[ПРОВАЛ]/[CRIT_SUCCESS]/[CRIT_FAIL]/[SUCCESS_COST] для проверок навыков. Шаг 6. ИИ получает числа и генерирует полный ответ: нарративное описание происходящего + DDS-команды для анимаций, эффектов, звуков, освещения, построек. Шаг 7. Сервер парсит DDS-команды из ответа: запускает анимации на 3D-сцене, проигрывает звуки, создаёт или удаляет блоки, меняет освещение, выводит нарративный текст в Чат №1. Фильтрация действий от вопросов в Чате №1. Сервер анализирует текст перед отправкой ИИ. Если текст заканчивается знаком «?» или содержит вопросительные слова (как, почему, где, когда, зачем, что если, сколько, какой, кто, куда, откуда) — сервер помечает запрос как [ВОПРОС]. ИИ отвечает ТОЛЬКО нарративом через /X без выполнения действий и без отдачи команды /R. Если текст не является вопросом — отправляется как [ДЕЙСТВИЕ] и обрабатывается полностью. Ввод игрока без изменений. Сервер отправляет текст ИИ ровно в том виде в каком он был введён. Сервер не исправляет орфографические ошибки, не подменяет слова синонимами, не пытается угадать что игрок имел в виду. ИИ — языковая модель которая понимает опечатки, синонимы, разговорные формулировки и сленг, поэтому нормализация не требуется. Подтверждение траты ресурсов. Если игрок описывает действие которое требует траты ограниченного ресурса (ячейка заклинания, дорогой компонент, одноразовый предмет) и при этом нет непосредственной угрозы, сервер показывает диалоговое окно: «Вы хотите потратить ячейку 3-го круга на это заклинание? Да / Нет». В бою подтверждение не запрашивается. Это предотвращает случайную трату ценных ресурсов. Обработка невозможных действий. Если игрок пытается сделать что-то физически невозможное — «прыгаю через пропасть шириной 50 метров», «поднимаю одной рукой каменную глыбу размером с дом», «лечу без крыльев и заклинания полёта» — ИИ отвечает в Чате №1: «Твой персонаж понимает что это невозможно. [Конкретная причина]. Попробуй другое действие.» Сервер даёт игроку дополнительные 15 секунд сверх обычного таймера хода. Если игрок второй раз подряд пытается сделать невозможное — его ход пропускается с сообщением «Ход пропущен: дважды выбрано невозможное действие.» Градация результатов проверок навыков: · Катастрофа (CRIT_FAIL): результат броска ниже DC на 10 или больше. Полный провал с дополнительными негативными последствиями. · Провал (FAIL): результат ниже DC на 1-9. Действие не удалось но без серьёзных последствий. · Успех с осложнением (SUCCESS_COST): результат ровно равен DC. Действие удалось но с каким-то осложнением или ценой. · Успех (SUCCESS): результат выше DC на 1-9. Действие удалось как задумывалось. · Триумф (CRIT_SUCCESS): результат выше DC на 10 или больше. Исключительный успех с дополнительными бонусами. Особые правила бросков: натуральная 1 (на кубике выпало 1) — всегда как минимум FAIL, даже если модификаторы дали бы результат выше DC. Натуральная 20 (на кубике выпало 20) — всегда CRIT_SUCCESS, даже если DC был выше любого возможного броска. Групповые проверки. Когда несколько персонажей делают что-то вместе (например вся партия крадётся или ищет скрытую дверь), сервер бросает d20 за каждого участника отдельно. Группа проходит проверку если половина или больше участников достигли успеха. CRIT_SUCCESS считается за два успеха. CRIT_FAIL считается за ноль успехов И даёт помеху на следующую проверку ближайшему союзнику. Что сервер считает локально без участия ИИ: · Бросок d20 и прибавление модификаторов характеристик и бонуса мастерства. · Сравнение результата с AC цели (для атак) или DC (для проверок навыков). · Бросок всех типов кубов урона (d4, d6, d8, d10, d12, d20, d100) и прибавление модификаторов. · Бросок инициативы для всех участников боя и сортировка порядка ходов. · Бросок спасбросков всех типов. · Бросок концентрации когда заклинатель получает урон во время поддержания заклинания. · Проверка морали врагов: сравнение текущего HP с порогом бегства. · Применение состояний: уменьшение счётчиков длительности, обработка эффектов в начале и конце хода. · Отслеживание ячеек заклинаний и их восстановление при отдыхе. Что всегда определяет ИИ и никогда сервер: · Классификацию действия игрока (что именно хотел сделать игрок). · Сложность (DC) и тип проверки для нестандартных действий. · Нарративное описание всего происходящего. · Какие DDS-команды нужно отправить для визуализации сцены. · Отыгрыш NPC: диалоги, реакции, эмоции, решения. · Генерацию новых локаций и размещение объектов. --- 1.3.1 Протокол DDS (Dungeons & Dragons Script) DDS это компактный мини-язык придуманный специально для общения между ИИ и сервером игры Dung Eons & AI. Каждая команда представляет собой короткую текстовую строку длиной от трёх до десяти символов. Формат любой команды: /PXY... где / — маркер начала строки, P — буква типа команды от A до Z, XY — параметры зависящие от типа. Строгий конечный автомат парсинга DDS. Когда сервер получает ответ от ИИ, он обрабатывает его строго построчно по следующим правилам: 1. Строка начинающаяся с /X — это нарративный текст. Сервер выводит его в Чат №1 для игроков. 2. Строка начинающаяся с / и следующей буквы от A до W или от Y до Z (любая кроме X) — это DDS-команда. Сервер исполняет её. 3. Строка НЕ начинающаяся с / — сервер записывает её в отладочный журнал и полностью игнорирует. Игрокам не показывается. 4. Пустая строка означает конец ответа ИИ. Всё что идёт после пустой строки в этом же ответе не обрабатывается. Валидация DDS-команд. Сервер проверяет каждую команду на соответствие трём критериям: длина строки должна быть от 3 до 10 символов включительно, все символы в параметрах должны быть допустимыми (только буквы латинского алфавита a-z и A-Z, и цифры 0-9), тип команды (буква после /) должен быть известен серверу из списка поддерживаемых команд. Если команда не проходит валидацию, сервер возвращает ИИ сообщение [DDS_ERROR] с указанием какая именно команда невалидна. ИИ обязан прислать исправленную команду того же типа. У ИИ есть 3 попытки на исправление. После трёх провалов сервер применяет fallback: показывает базовую анимацию idle, создаёт слабую белую вспышку, и отправляет алерт разработчику. Версионирование DDS. В system-сообщении которое отправляется ИИ при каждом запросе содержится строка: DDS_VERSION=1 и SUPPORTED_COMMANDS=A,E,S,B,K,T,V,Vs,D,C,M,P,X,R,W,Wf,H,N,I,Y,Q,G,F,Xi,Z,L,U,Cl. Это позволяет серверу и ИИ быть уверенными что они используют одну и ту же версию протокола. Игроки НИКОГДА не видят сырые DDS-команды. Все команды вроде /Ba23X5YA или /Aja1 обрабатываются сервером и преобразуются в визуальные эффекты, анимации, постройки. Игрок видит только конечный результат и краткое текстовое описание. Компонент 1 — Что именно движется, меняется или разрушается. ИИ перечисляет объекты или блоки участвующие в действии. Для персонажа: «голова», «тело», «правая рука», «левая рука», «правая нога», «левая нога», «все блоки». Для объектов на сцене: «блоки стены», «блоки колонны», «блок по координатам X5Y3», «все блоки в радиусе 3 от точки X0Y0». Для разрушений: «блоки типа b в области X2Y2-X5Y5» (стены в прямоугольной области). Компонент 2 — Куда или как двигаются. Направление: «вверх», «вниз», «вперёд», «назад», «влево», «вправо». Степень: «на N градусов», «на N клеток», «до координат X#Y#». Для разрушений: «рассыпаются на N обломков», «падают в направлении X», «во все стороны на N клеток». Компонент 3 — Характер движения. «мгновенно» (без анимации), «плавно за N секунд», «дрожа с частотой N», «по дуге радиусом N», «циклично N раз», «пока не столкнётся с [объект]». Для разрушений: «сначала трескаются за 0.5с, затем обрушиваются за 2с», «взрываются мелкими обломками за 1с». Компонент 4 — Изменение цвета или материала. «без изменения цвета», «меняется на [код цвета из палитры a-z]», «пульсирует между [код1] и [код2] с периодом N секунд», «становится прозрачным на N процентов», «покрывается трещинами». Компонент 5 — Точка отсчёта для движений. «относительно своей обычной позиции» (стандартное положение этой части тела), «относительно текущей позиции» (где блок находится сейчас), «относительно [другой блок или объект]», «относительно нулевых координат мира». Вы не нажимаете кнопки «атаковать» или «использовать способность» — вы описываете словами что именно делает ваш персонаж. «Я замахиваюсь мечом и бью орка по голове, стараясь пробить его шлем» вместо нажатия кнопки. «Я крадусь вдоль стены оставаясь в тени, и внимательно смотрю под ноги чтобы не наступить на ловушку» вместо зажатия клавиши Shift. ИИ читает ваше описание, интерпретирует его по правилам D&D, определяет какие проверки нужны, сервер бросает виртуальные кубики, и ИИ красочно описывает что получилось. Такой подход максимально приближает цифровую игру к опыту живой партии за столом, где ваше воображение, красноречие и умение описывать действия важнее скорости нажатия кнопок. Ведущий должен потратить многие часы на подготовку приключения, изучить сотни страниц правил, уметь импровизировать, быть беспристрастным арбитром, и регулярно выделять время на проведение игры. Из-за этого множество групп годами не могут начать играть. ИИ-ведущий решает эту проблему радикально. ИИ не устаёт и может вести игру 24 часа в сутки 7 дней в неделю. ИИ абсолютно беспристрастен — у него нет любимчиков, он не подсуживает друзьям, он применяет правила строго и одинаково ко всем игрокам. ИИ помнит все правила выбранной редакции досконально — он никогда не забудет редкое правило про концентрацию заклинаний под водой, не перепутает модификаторы укрытия, не ошибётся в расчёте урона от падения. ИИ способен вести тысячи партий одновременно — это позволяет игре масштабироваться на неограниченную аудиторию без необходимости нанимать и обучать живых ведущих. С технической точки зрения, один прокси-сервер с цепочкой из четырёх бесплатных API способен обслуживать до 430 активных игроков ежедневно без каких-либо финансовых затрат на ИИ-инфраструктуру. При росте аудитории достаточно добавить новые API в цепочку или перейти на платные тарифы существующих провайдеров для увеличения лимитов. 2.3 Блоковая система мира Весь визуальный мир Dung Eons & AI состоит из кубических блоков. Фундаментальный размер одного блока — ровно половина метра на половину метра на половину метра. Из этих кубиков как из детского конструктора собирается абсолютно всё: каменные стены подземелий, деревянные полы таверн, кованые железные двери, дубовые столы и стулья, вековые деревья в лесу, скальные выступы в горах, и даже сами персонажи игроков и неигровые персонажи — их тела, конечности, головы представляют собой конструкции из блоков разного размера и цвета. На одной сцене может находиться до 25 000 блоков одновременно. Реальное количество зависит от размера и сложности локации: маленькая комната в таверне состоит из нескольких сотен блоков, а эпическая тронная зала с колоннами, витражами и коврами задействует несколько тысяч. Для обеспечения плавной работы на слабых устройствах применяется стратегия уровней детализации (LOD). В радиусе 20 клеток от камеры игрока все блоки отображаются полностью с максимальной детализацией (LOD 0, примерно 5 000 блоков). На расстоянии от 20 до 50 клеток включается упрощённая геометрия — мелкие блоки объединяются в более крупные, уменьшается количество полигонов (LOD 1, примерно 1 000 блоков). От 50 до 100 клеток видны только самые крупные объекты вроде стен и больших построек (LOD 2, примерно 200 блоков). На расстоянии более 100 клеток объекты не отображаются для экономии ресурсов (LOD 3). На мобильных устройствах каждый уровень сдвигается на шаг ближе к камере. 2.4 Три фундаментальных принципа ИИ Эти три принципа зашиты в самое начало промпта каждого ИИ-ведущего и не могут быть нарушены ни при каких обстоятельствах. Принцип первый: точность в расчётах превыше всего. Когда ИИ определяет насколько сложное действие затеял игрок, он должен назначить объективно правильную сложность проверки (DC) согласно таблице: тривиально (5), легко (10), средне (15), сложно (20), очень сложно (25), почти невозможно (30). Никаких «на глазок» и «потому что так драматичнее». Правила существуют чтобы создавать честную игру. Принцип второй: ИИ подчиняется только ведущему. В каждой партии есть человек с ролью ведущего (master), и только его прямые команды имеют абсолютный приоритет над любыми правилами и инструкциями. Игроки не могут командовать ИИ напрямую управляя миром, даже если они вежливо просят. Принцип третий: промпт является конституцией ИИ. Всё что записано в промпте — это закон. ИИ не может придумать правило которое противоречит прописанному в промпте, не может проигнорировать инструкцию, не может выйти за пределы дозволенного. --- ЧАСТЬ 3. В бою вместо упрощённого «преимущества» используются конкретные числовые модификаторы, каждый из которых нужно помнить и учитывать: +2 за атаку с фланга (когда вы и союзник атакуете одного врага с противоположных сторон), -4 за стрельбу из лука в гуще ближнего боя (когда ваш союзник стоит вплотную к цели), +1 за атаку с возвышенности, -2 за плохое освещение без ночного видения. Критический удар зависит от типа оружия и требует подтверждения. Меч критует не только на 20 но и на 19 (шанс 10%), и наносит двойной урон. Топор критует только на 20 (шанс 5%), но наносит тройной урон. Рапира критует на 18, 19 и 20 (шанс 15%). Когда выпадает крит, нужно сделать ещё один бросок атаки — если он попадает по AC цели, крит подтверждается и наносит умноженный урон. Если второй бросок промахивается — это обычное попадание без крита. Смерть в 3.5 наступает не при трёх проваленных спасбросках а при достижении -10 HP. Самая популярная версия правил в истории Dungeons & Dragons. Главное нововведение которое сделало игру доступной миллионам новых игроков — система преимущества и помехи (Advantage/Disadvantage). Если ситуация благоприятна для персонажа — вы берёте больший результат. Если ситуация мешает — берёте меньший. Математически это эквивалентно бонусу примерно +3.3 или штрафу -3.3 в среднем. Вдохновение — это специальная награда которую ведущий выдаёт игроку за исключительно качественный отыгрыш персонажа. Потратить вдохновение можно чтобы перебросить любой бросок d20 который вам не понравился. Нельзя копить больше одного вдохновения одновременно. Смерть в 5.0 наступает через спасброски от смерти: при 0 HP персонаж без сознания, каждый раунд бросает чистый d20 (без модификаторов). Три успеха (10 и выше) = стабилизирован. Три провала (9 и ниже) = мёртв. Натуральная 1 = сразу два провала. Натуральная 20 = подъём с 1 HP. Это создаёт драматические моменты когда судьба персонажа висит на волоске в течение нескольких раундов, и союзники могут попытаться спасти его. Редакция 5.5 (2024) — осовремененная и исправленная. Эта версия вобрала десять лет обратной связи от сообщества и исправила основные недостатки пятой редакции. Базовое заклинание лечения (Cure Wounds) теперь восстанавливает 2d8 хитов вместо 1d8 на первом уровне, что делает начальные уровни менее смертельными. Идеально для тех кто хочет тестировать билды и сражаться без остановки.» При создании кампании также настраиваются следующие параметры: · Редакция правил: выбор между 3.5, 5.0 (2014) и 5.5 (2024). Влияет на все правила по которым будет работать ИИ-ведущий. · Пророчество: включено или выключено. Если включено, ИИ сгенерирует в начале кампании туманное предсказание из 4-6 строк которое будет постепенно сбываться по ходу сюжета. · Максимальное количество игроков: от 1 до 10. · Ведущий: кто из присутствующих игроков будет иметь права ведущего. РЕЖИМ «ВЕДУЩИЙ» И РЕЖИМ «ВЕДУЩИЙ + ИИ» Режим «Ведущий» — полное описание В обычном режиме игры ведущий это такой же участник как и все — он управляет своим персонажем через Чат №1. Но когда активирован режим «Ведущий», у этого человека появляется дополнительная панель инструментов в левой части экрана, которая позволяет напрямую редактировать игровой мир. Панель содержит шесть вкладок. Вкладка «Бестиарий». Открывает полный список всех существ из выбранной редакции D&D — от крошечных крыс до титанов и божеств. Для каждого существа показаны: все шесть характеристик со значениями и модификаторами, кости хитов и максимальное здоровье, класс брони и из чего он складывается, все атаки с бонусами попадания и костями урона, особые способности с полным описанием как они работают, сопротивления и уязвимости к разным типам урона. Ведущий может выбрать любое существо, изменить любые его параметры — например превратить обычного гоблина с 7 HP в гоблина-вожака с 45 HP, дополнительной атакой и сопротивлением к ядам — и одним кликом разместить его в любом месте текущей локации. Вкладка «Предметы». Конструктор экипировки. Ведущий создаёт оружие (выбирает тип, кость урона, свойства вроде лёгкого или фехтовального, магические бонусы к атаке и урону), броню (тип, базовый AC, требования по Силе, магические свойства), аксессуары (кольца, амулеты, пояса с уникальными эффектами вроде регенерации или сопротивления), расходники (зелья, свитки, яды). Для каждого предмета задаётся редкость (от Обычного до Легендарного) и минимальный уровень для использования. Вкладка «Локации». Редактор сцен где ведущий может размещать блоки и шаблоны, менять освещение, выбирать фоновую музыку. Можно использовать готовые шаблоны (таверна, храм, подземелье) и модифицировать их, или строить полностью с нуля. Вкладка «Сюжет». Позволяет задавать события — пары «условие → действие». Условие это триггер который сервер проверяет после каждого хода: «когда игроки входят в тронный зал», «когда HP босса падает ниже 50%», «когда игрок-жрец касается алтаря». Действие это набор DDS-команд которые исполняются при срабатывании: показать описание, запустить анимацию, телепортировать персонажа, сменить музыку. Вкладка «NPC». Создание неигровых персонажей с именем, расой, классом, характером (из 8 типов), отношением к каждому игроку (0-100), скрытой историей которая будет раскрываться постепенно. Вкладка «Консоль». Прямой текстовый чат с ИИ где ведущий пишет запросы на генерацию контента. «Придумай босса для подземелья пятого уровня», «Создай легендарный меч с проклятием жадности для седьмого уровня», «Опиши заброшенный эльфийский храм в осеннем лесу». ИИ немедленно генерирует ответ со всеми необходимыми DDS-командами. Режим «Ведущий + ИИ» — полное описание Этот режим предназначен для ведущих которые хотят контролировать повествование не отнимая время у игроков. Он добавляет этап проверки между генерацией ответа ИИ и показом его игрокам. Процесс работает так: игрок отправляет действие, сервер передаёт его ИИ, ИИ генерирует ответ (нарратив + DDS-команды). НО вместо немедленной отправки игрокам, ответ появляется в специальном маленьком окне предпросмотра в углу экрана ведущего. Заголовок окна: «Предпросмотр ответа ИИ». Таймер в окне показывает оставшееся время на принятие решения (по умолчанию 5 секунд, настраивается от 3 до 10). За эти 5 секунд ведущий может выполнить одно из четырёх действий: 1. Ничего не нажать — через 5 секунд ответ автоматически уходит игрокам в исходном виде. 2. Нажать кнопку «✓ ОК» — ответ уходит немедленно не дожидаясь окончания таймера. 3. Нажать кнопку «✎ Изменить» — открывается редактор где ведущий может исправить текст описания, добавить или удалить DDS-команды. После редактирования изменённый ответ уходит игрокам. 4. Нажать кнопку «✗ Отклонить» — ведущий пишет короткое пояснение (например «добавь больше драмы», «этот орк не должен умирать от одного удара», «опиши эффект огня ярче»), и ИИ перегенерирует ответ с учётом замечания. 5. Нажать кнопку «↺ Заменить» — ведущий полностью пишет свой вариант ответа который заменяет всё что сгенерировал ИИ. Этот режим критически важен для сюжетных кампаний где ведущий выстраивает сложное повествование и не хочет чтобы ИИ случайной импровизацией сломал ключевой сюжетный момент. Система «Сломано» — ответственность ведущего. С большой властью приходит ответственность. Если ведущий злоупотребляет инструментами и разрушает игровой баланс — например выдаёт легендарное оружие (+3, аура защиты, полёт) персонажу первого уровня, или делает всех врагов бессмертными, или даёт двадцатый уровень новичку — сервер присваивает партии статус «Сломано». Статус отображается в профиле кампании и виден другим игрокам. Последствия статуса «Сломано»: за достижения совершённые в этой партии не начисляются монетки для главного меню, партия исключается из глобальных рейтингов и топов, персонажи из этой партии не могут быть опубликованы в Мастерской. Снять статус можно только исправив все нарушения баланса и отправив запрос на перепроверку сервером. --- ЧАСТЬ 6. БЕСТИАРИЙ И АРТЕФАКТЫ 6.1 Бестиарий Бестиарий — это полное собрание всех существ встречающихся в мире Dungeons & Dragons выбранной редакции. Бестиарий зашит в код сервера и полностью передаётся ИИ в составе system-сообщения (промпта), чтобы ИИ мог в любой момент обратиться к характеристикам любого существа не придумывая их на ходу. Для каждого существа в бестиарии указана следующая информация: все шесть базовых характеристик (Сила, Ловкость, Телосложение, Интеллект, Мудрость, Харизма) с числовыми значениями; тип и количество костей хитов определяющих максимальное здоровье; класс брони (AC) и его источники (природная броня, ловкость, магические бонусы); все атаки с бонусами попадания, костями урона и типом урона; особые способности с полным описанием механики (например «Драконье дыхание: конус 30 футов, урон 2d6 огнём, спасбросок Ловкости DC 13 для половины урона, перезарядка 5-6»); сопротивления и уязвимости к различным типам урона; порог морали — при каком проценте здоровья существо попытается сбежать; размер существа (Маленький, Средний, Большой, Огромный, Колоссальный); скорость передвижения в футах; и блочная модель — из каких блоков состоит визуальное представление существа на сцене. ИИ знает полный список оружия из правил D&D (все 26 типов оружия с костями урона, типом урона и свойствами). ИИ также может создавать новое уникальное оружие не описанное в официальных книгах правил, используя специальный код z в DDS-командах атаки. При использовании этого кода ИИ должен описать характеристики оружия в нарративе. 6.2 Артефакты Артефакты — это особые магические предметы обладающие уникальными свойствами. Их нельзя купить в обычной оружейной лавке или заказать у кузнеца. Артефакты находят в древних сокровищницах, получают в награду от королей и архимагов, или отбирают у поверженных врагов. Как надеть или снять артефакт. Игрок НЕ может сделать это через интерфейс инвентаря перетаскиванием предмета в слот. Для надевания или снятия артефакта нужно написать действие в Чат №1: «надеваю кольцо невидимости на указательный палец правой руки», «снимаю проклятый амулет и бросаю его в реку», «надеваю эльфийский плащ застёгивая фибулу». ИИ классифицирует это как взаимодействие (код /Rh) и описывает результат. Это сделано для погружения: персонаж не может мгновенно сменить кольцо посреди боя через удобный интерфейс, ему нужно потратить действие. Категории артефактов: · Оружие: мечи, топоры, луки, посохи и любое другое оружие с магическими свойствами. Может давать бонусы к атаке и урону, дополнительный урон стихией (огнём, холодом, молнией), особые способности вроде свечения при приближении орков или возможности раз в день применить заклинание. · Броня: щиты, нагрудники, шлемы с магической защитой. Дают бонус к AC, сопротивление к определённым типам урона, ауры которые защищают союзников поблизости. · Аксессуары: кольца, амулеты, пояса, плащи, диадемы. Повышают характеристики, дают защиту от состояний (нельзя испугать, нельзя отравить), обеспечивают регенерацию здоровья. · Расходники: зелья, свитки, эликсиры — используются один раз и исчезают. · Проклятые предметы: выглядят как полезные артефакты но несут скрытое негативное свойство. Например кольцо которое носитель не может снять без заклинания Remove Curse, или меч который заставляет владельца атаковать ближайшее существо независимо от того друг это или враг. Редкость артефактов определяет их силу и минимальный уровень персонажа для использования: · Обычный (Common): +1 к AC или +1 к урону. Можно использовать с 1 уровня. · Необычный (Uncommon): +1 к двум разным характеристикам или сопротивление одному типу урона. С 3 уровня. · Редкий (Rare): +2 к атаке, дополнительный урон 2d6 огнём/холодом/молнией. С 6 уровня. · Очень редкий (Very Rare): +2 к атаке, возможность раз в день бесплатно применить сильное заклинание. С 11 уровня. · Легендарный (Legendary): +3 к атаке, аура защиты на 10 футов вокруг, способность превращения. С 17 уровня. Персонаж может быть настроен (attuned) максимум на три артефакта одновременно. --- ЧАСТЬ 7. УПРАВЛЕНИЕ ПЕРСОНАЖЕМ Процесс управления персонажем в Dung Eons & AI принципиально отличается от традиционных видеоигр и требует от игрока другого подхода. Вместо того чтобы нажать кнопку «Атаковать» и смотреть как персонаж бьёт врага, игрок описывает в Чате №1 что именно делает его персонаж. Сервер отправляет этот текст ИИ вместе с полным контекстом игры (локация, позиции всех существ, характеристики, эффекты). ИИ читает описание, понимает что игрок хочет сделать, и классифицирует действие через DDS-команду /R с указанием типа действия. Например игрок пишет: «Я делаю обманный выпад влево и когда гоблин открывается для контратаки — резко бью мечом по его правому боку целясь под рёбра». ИИ анализирует это описание и определяет что это атака (/Rl), и поскольку игрок описывает хитрый финт а не просто машет мечом, ИИ может дать преимущество на этот бросок. Сервер получив классификацию делает все необходимые броски: d20 + модификатор Силы (для атаки мечом) + бонус мастерства. Если преимущество — бросает d20 дважды и берёт больший результат. Сравнивает с AC гоблина. Если попадание — бросает кубы урона (d8 для длинного меча) и прибавляет модификатор Силы. Результаты отправляются ИИ в структурированном формате: [A] Алан: атака 19 (d20=12 +7) против AC 15 — попадание и [Урон] 8 рубящего (d8=5 +3). ИИ генерирует красочное описание: «Твой финт срабатывает идеально — гоблин покупается на обманный выпад и открывается. Ты резко переносишь вес и вонзаешь меч ему под рёбра. Лезвие пробивает кожаную броню, из раны толчками вытекает тёмная кровь. Гоблин хрипит и отшатывается, его движения становятся неуверенными — он серьёзно ранен.» Детальные и творческие описания действий активно поощряются игрой. Если вместо «атакую» игрок описывает конкретную технику, цель на теле врага, обстоятельства — ИИ может дать преимущество на бросок (бросить d20 дважды и взять лучший результат) или снизить DC проверки на 1-2 пункта. Однако есть и ограничения. Игрок не может сделать физически невозможное. Не может использовать способности и заклинания которых у его персонажа нет. Не может управлять миром как ведущий. --- ЧАСТЬ 8. МЕХАНИКИ DUNGEONS & DRAGONS Сервер Dung Eons & AI реализует все ключевые механики D&D без участия ИИ — это сделано для скорости и точности. Спасброски. Когда персонаж пытается избежать опасного эффекта — увернуться от драконьего дыхания, устоять перед ядовитым газом, не поддаться магическому очарованию — он делает спасбросок. Бросается d20, прибавляется модификатор соответствующей характеристики, результат сравнивается с DC (сложностью) эффекта. При успехе урон снижается вдвое или эффект полностью игнорируется. Концентрация. Некоторые мощные заклинания требуют от заклинателя постоянного сосредоточения — например невидимость союзника, полёт, или призыв магического оружия. Заклинатель может поддерживать только одно такое заклинание одновременно. Если заклинатель получает урон пока концентрируется, он должен пройти спасбросок Телосложения. DC = 10 или половина полученного урона — смотря что больше. При провале концентрация теряется и заклинание немедленно прекращается. Визуально концентрация отображается как тонкая светящаяся нить между заклинателем и поддерживаемым эффектом. Отдых. Короткий отдых длится 1 час и позволяет потратить кости хитов для восстановления здоровья. Длинный отдых — это полноценный ночной сон в течение 8 часов (минимум 6 часов сна), после которого полностью восстанавливаются хиты, все ячейки заклинаний, и половина потраченных костей хитов. Длинным отдыхом можно пользоваться не чаще раза в 24 игровых часа. Спасброски от смерти. Когда персонаж падает до 0 HP, он теряет сознание и больше не может действовать, использовать бонусные действия или реакции. В начале каждого своего хода он бросает d20 без модификаторов. Три успеха (10 или выше) — персонаж стабилизируется, HP остаётся 0 но он жив. Три провала (9 или ниже) — персонаж умирает. Натуральная 1 считается за два провала сразу. Натуральная 20 — персонаж приходит в сознание с 1 HP. ГЕНЕРАЦИЯ ВНЕШНОСТИ ПЕРСОНАЖА Внешность персонажа в Dung Eons & AI это не просто статичная аватарка, а детальная блочная конструкция собранная из множества кубиков разного размера и цвета. Базовое телосложение персонажа определяется одним из шести пресетов, и этот выбор существенно влияет на то как персонаж выглядит и как на нём сидит одежда. Оружие в правой и левой руке с полными характеристиками: название, кость урона, тип урона, свойства (лёгкое, фехтовальное, двуручное, метательное и так далее), магические бонусы если есть. Надетая броня с её классом брони и свойствами. Три слота настроенных артефактов с указанием названия, редкости и эффекта. Расходные предметы с указанием количества: зелья лечения (сколько штук), свитки заклинаний, яды для оружия, алхимические бомбы. Обычные предметы: верёвка, факелы, инструменты, запасная одежда, провизия. И деньги: золотые монеты, серебряные, медные. Вкладка «Внешность и предыстория» разделена на две секции. Верхняя секция показывает текущую внешность персонажа: выбранный пресет телосложения со словесным описанием, цвет кожи, цвет и стиль волос, цвет глаз, особые приметы вроде шрамов или татуировок, и перечень всех надетых предметов одежды с их цветами. Эта секция обновляется автоматически когда игрок меняет экипировку через Чат №1. Нижняя секция показывает предысторию: черты характера, идеалы, привязанности, слабости и недостатки. Эта секция заполняется один раз при создании персонажа и более никогда не меняется. Вкладка «Связи» показывает социальное положение персонажа в мире. Отношения с фракциями отображаются в виде полосок от 0 до 100 с текстовой меткой: «Гильдия Торговцев: 45 (нейтрально) — стандартные цены», «Храм Света: 72 (дружелюбие) — бесплатное лечение», «Гильдия Воров: 15 (враждебность) — возможны нападения». Список известных NPC с которыми персонаж взаимодействовал — для каждого указано текущее отношение и краткая заметка о последней встрече. Активные квесты с описанием цели, обещанной награды и срока выполнения. Завершённые квесты для истории. И текущее пророчество если оно есть в кампании. --- ЧАСТЬ 15. ТРИ ЧАТА Три чата являются основным средством коммуникации в игре. Они располагаются в левом нижнем углу экрана поверх 3D-сцены, организованные вертикальным столбиком. Ключевой порядок: самый нижний чат это Чат №1, над ним Чат №2, и над ним Чат №3. Порядок снизу вверх: 1 → 2 → 3. Чаты занимают примерно тридцать процентов ширины экрана и около сорока процентов высоты. Фон каждого чата тёмный полупрозрачный с эффектом размытия чтобы сквозь них была видна игра но текст оставался читаемым. По умолчанию в каждом чате отображаются последние два сообщения; если сообщение слишком длинное оно обрезается с многоточием. Чат №1 — Игровой чат. Это главный интерфейс взаимодействия с миром. Здесь отображается нарратив от ИИ (белым цветом с префиксом [ИИ]), действия игроков (у каждого игрока свой уникальный цвет, с префиксом [Имя]), результаты бросков кубиков (жёлтым цветом, дублируются крупным текстом в центре экрана на три секунды с анимацией появления), системные сообщения (серым цветом с иконкой свитка). DDS-команды отображаются в человекочитаемом виде: сервер преобразует технические команды в текст понятный игроку. Чат №2 — Чат вопросов к ИИ. Этот чат ПОДКЛЮЧЕН К ИСКУССТВЕННОМУ ИНТЕЛЛЕКТУ и предназначен для технических вопросов о механике, правилах, расчётах, способностях. Сообщения отсюда отправляются ИИ с пометкой [ВОПРОС ВНЕ ИГРЫ], и ИИ отвечает описательно не производя никаких игровых действий. Ответы видны только спросившему игроку. Чат №3 — Общий чат. ПОЛНОСТЬЮ ИЗОЛИРОВАН от ИИ. Сообщения никогда не отправляются в API. Свободное общение без фильтров. Голосовой чат работает по принципу рации (нажал кнопку — говоришь) и не имеет фильтров. Громкость голоса влияет на дальность слышимости: тихий голос — радиус три клетки, нормальный — вся комната, крик — комната плюс соседние. Стены блокируют звук на основе геометрии локации. Тактический стол это инструмент для детального планирования, не заменяющий общение. При активации сцена замирает, в центре появляется стол с миниатюрой видимой части локации где игроки могут двигать фигурки, рисовать стрелки и линии, использовать несколько уровней высоты для демонстрации планов полёта или подкопа. --- ЧАСТЬ 16. СЛЫШИМОСТЬ И РАССТОЯНИЕ Механика слышимости в Dung Eons & AI работает на трёх факторах: громкость речи, расстояние до слушателя, и наличие препятствий между говорящим и слушателем. Громкость речи определяется способом коммуникации. При текстовом вводе игрок явно указывает громкость через то как он формулирует сообщение. Если игрок пишет «шепчу: ...» или «тихо говорю: ...» — это задаёт радиус слышимости в 3 клетки. Обычное сообщение без специальных указаний громкости считается речью и слышно всем в текущей комнате. Если игрок пишет «кричу: ...» или «ОРУ: ...» — это слышно в текущей комнате плюс во всех соседних комнатах. При голосовом общении сервер измеряет среднюю громкость микрофона за последние две секунды и определяет категорию: тихо, нормально, или громко. Расстояние вычисляется как евклидово расстояние между координатами говорящего и слушателя в клетках. Если расстояние превышает радиус слышимости для текущей громкости, звук не доходит до слушателя совсем. Препятствия проверяются через карту блоков которая передаётся в контексте каждого запроса. Сервер проводит воображаемый луч от позиции говорящего к позиции слушателя. Если этот луч пересекает блок с типом «стена» или блок с типом «дверь» в состоянии «закрыта», громкость снижается на одну категорию: шёпот через стену полностью блокируется и не слышен совсем, обычная речь через стену становится приглушённым шёпотом (радиус 3 клетки с той стороны), крик через стену приглушается до обычной речи. Если луч пересекает две стены подряд, любая речь полностью блокируется. Отдельный случай — подслушивание разговоров. Когда два NPC общаются между собой без участия игрока, над их головами отображается текст их разговора. Игрок может читать этот текст просто глядя на NPC — для этого не нужно никаких специальных действий, текст виден всем кто находится в радиусе слышимости. Но если игрок хочет подслушать ТО ЧЕГО ЕМУ СЛЫШАТЬ НЕ ПОЛАГАЕТСЯ (например разговор за закрытой дверью, или шёпот в углу таверны), в дело вступает проверка Скрытности игрока против Внимательности NPC. Если игрок проходит проверку — он слышит разговор. Если проваливает — NPC замечают слежку и реагируют соответственно своему характеру: дружелюбный NPC может просто замолчать и сменить тему, ворчливый может громко возмутиться и привлечь внимание всей таверны, жестокий может атаковать. --- ЧАСТЬ 17. NPC И ОТЫГРЫШ Неигровые персонажи в Dung Eons & AI это полноценные личности со своим характером, памятью, отношением к партии и скрытыми историями. ИИ отыгрывает каждого NPC индивидуально основываясь на его характере, текущем отношении к игроку, и контексте ситуации. Система характеров NPC включает восемь типов. Дружелюбный персонаж улыбается при встрече, охотно предлагает помощь не ожидая награды, говорит открыто и доброжелательно, делает комплименты, прощает мелкие ошибки. Ворчливый персонаж постоянно жалуется — на жизнь, на налоги, на погоду, на молодёжь которая совсем распустилась, на правительство которое всё разворовало, на цены которые растут, на качество товаров которое падает. Но за внешней ворчливостью часто скрывается доброе сердце: ворчливый кузнец будет жаловаться что опять пришли какие-то авантюристы и отвлекают его от работы, но меч починит на совесть и плату возьмёт справедливую. Таинственный персонаж говорит загадками и намёками, никогда не раскрывает своих истинных мотивов, отвечает вопросом на вопрос, появляется и исчезает неожиданно. Трусливый персонаж боится всего — громких звуков, незнакомцев, темноты, оружия, магии, собак, лошадей, и особенно когда эти вещи сочетаются. Его легко запугать просто достав меч из ножен, но так же легко успокоить проявив доброту. Надменный персонаж смотрит на окружающих свысока, требует особого обращения, подчёркивает свой статус и происхождение, оскорбляется если к нему обращаются без должного почтения. Жестокий персонаж наслаждается чужими страданиями — он будет пытать пленника не для получения информации а ради удовольствия, он добьёт раненого врага а не возьмёт в плен, он установит несправедливо высокую цену просто потому что может. Преданный персонаж служит кому-то или чему-то — королю, божеству, кодексу чести, или конкретному человеку — и не предаст ни за какие деньги, угрозы или магию. Безумный персонаж непредсказуем в принципе — он может засмеяться посреди серьёзного разговора, заплакать от вида цветка, или внезапно разозлиться без видимой причины, и никто не знает что он сделает в следующую секунду включая его самого. Шкала отношений это число от 0 до 100 которое есть у каждого NPC к каждому игроку отдельно. Это не общее отношение к партии, а персональное к конкретному персонажу. От 0 до 20 это враждебность: NPC может напасть без предупреждения, отказать в помощи, намеренно навредить, натравить стражу, солгать чтобы подставить. От 21 до 40 это неприязнь: NPC будет неохотно помогать, завышать цены, грубить, отпускать едкие комментарии, делать всё возможное чтобы взаимодействие было неприятным. От 41 до 60 это нейтральное отношение: NPC относится к персонажу как к незнакомцу, без особых эмоций в ту или иную сторону, базовые услуги предоставляются по стандартным ценам. От 61 до 80 это дружелюбие: NPC поможет в разумных пределах, сделает скидку, поделится полезной информацией, может оказать небольшую услугу бесплатно. От 81 до 100 это преданность: NPC рискнёт жизнью ради персонажа, расскажет секреты которые не рассказал бы никому, станет постоянным союзником и пойдёт за персонажем в огонь и воду. Отношения меняются от действий игроков. Оскорбление снижает отношение на 10 пунктов у данного NPC. Помощь повышает на 15 пунктов. Героический поступок совершённый на глазах у NPC (например спасение его жизни или жизни его близких) повышает отношение на 25 пунктов. Мелкие действия вроде вежливого приветствия или небольшой чаевой могут сдвинуть отношение на 1-2 пункта. Отношения также могут меняться косвенно: если NPC слышит от других персонажей о подвигах или злодеяниях партии, его отношение корректируется даже без личного взаимодействия. Память NPC работает избирательно. Каждый NPC запоминает то что важно: имена персонажей с которыми он общался, данные ему обещания (особенно невыполненные), высказанные в его адрес угрозы или оскорбления, важные факты которые персонажи сообщили о себе, эмоциональный тон разговора (было ли общение приятным или конфликтным). Неважные детали NPC забывает: какая была погода во время разговора, во что был одет персонаж, точные формулировки проходных фраз, через сколько дней произошла встреча. Если партия пообещала кузнецу месяц назад принести редкую руду для особого заказа, и возвращается — кузнец спросит где руда, и его отношение упадёт на 10 пунктов если партия забыла про обещание. Если партия угрожала трактирщику расправой, при следующей встрече трактирщик немедленно пошлёт мальчишку за стражей, даже если партия заходит с улыбкой и деньгами. Скрытые истории есть не у всех NPC. Значимые персонажи — трактирщики, кузнецы, главы гильдий, квестодатели, ключевые NPC сюжета — имеют скрытую историю с вероятностью 60 процентов. Обычные NPC — уличные торговцы, стражники, слуги в замке — имеют скрытую историю с вероятностью 20 процентов. Фоновые персонажи — крестьяне на рынке, прохожие на улице, посетители таверны на заднем плане — имеют скрытую историю с вероятностью всего 5 процентов. Большинство фоновых персонажей действительно просто обычные люди без двойного дна. Трактирщик — это просто трактирщик который варит эль и сдаёт комнаты. Но конкретный крестьянин в деревне может оказаться замаскированным древним золотым драконом который выбрал тихую жизнь среди смертных. Конкретный нищий на паперти храма может быть бывшим архимагом который потерял память после неудачного эксперимента с запретной магией. Конкретная старуха-травница в лесу может быть последней из исчезнувшего народа который хранит тайну древнего пророчества. Эти скрытые истории раскрываются постепенно через намёки, зацепки, и развитие отношений с NPC. Скрытые сопротивления и уязвимости врагов никогда не показываются в интерфейсе явно. Игроки узнают о них только из описаний ИИ во время боя или исследования. Когда файербол мага ударяет в дракона а чешуя почти не обгорает — игроки понимают что у дракона сопротивление к огню. Когда серебряное оружие причиняет врагу гораздо больше страданий чем должно — становится ясно что у врага уязвимость к серебру. Когда заклинание холода заставляет слизь замедлиться и затвердеть по краям — уязвимость к холоду. Эта система заставляет игроков экспериментировать с разными типами урона и внимательно читать описания вместо того чтобы полагаться на цифры в интерфейсе. --- ЧАСТЬ 18. БОЙ И ИНИЦИАТИВА Когда начинается бой — засада гоблинов в пещере, встреча с патрулём стражи, или эпическое сражение с боссом в конце подземелья — все участники делают бросок инициативы. Каждый персонаж и каждое существо бросает двадцатигранный кубик (d20) и прибавляет свой модификатор Ловкости. Сервер собирает все результаты вместе и сортирует их от самого высокого к самому низкому. Этот порядок фиксируется на весь бой и определяет кто за кем ходит. Если у нескольких участников выпали одинаковые значения, порядок между ними определяется по дополнительному броску d20 без модификаторов (переброс до разрешения ничьей). В нижней панели персонаж который сейчас ходит подсвечивается яркой зелёной рамкой которая мягко пульсирует привлекая внимание. Персонаж который будет ходить следующим подсвечивается жёлтой рамкой. Все остальные участники не имеют рамки. Рядом с иконкой активного персонажа отображается круговой таймер: полный круг соответствует тридцати секундам которые даются на принятие решения и ввод действия. Круг постепенно сжимается, и когда остаётся пять секунд, он начинает пульсировать красным а звуковой сигнал напоминает что нужно поторопиться. Если игрок не успевает ввести действие за тридцать секунд, его персонаж автоматически пропускает ход (стоит на месте в защитной стойке). Для игроков в режиме новичка таймер увеличен до шестидесяти секунд. Мораль врагов моделирует их готовность продолжать бой когда ситуация складывается не в их пользу. Гоблины и кобольды это трусливые существа которые бросаются врассыпную как только понимают что проигрывают — когда их здоровье падает ниже половины, они немедленно пытаются убежать, используя действие Отход чтобы не провоцировать атаку. Бандиты и наёмники более стойкие: они дерутся пока здоровье не упадёт до четверти, потому что им платят за риск, но они не самоубийцы и не готовы умирать за хозяйское добро. Дикие звери — волки, медведи, пантеры — отступают когда ранены примерно на треть здоровья, если только они не защищают детёнышей или логово. Конструкты (големы, элементали) и нежить (скелеты, зомби, призраки) полностью лишены инстинкта самосохранения — они не чувствуют боли и страха, и будут сражаться пока их физически не разрушат до последнего хита. Религиозные фанатики и берсерки впадают в боевое безумие и отступают только когда находятся на пороге смерти — на десяти процентах здоровья или даже меньше. Интерактивное окружение позволяет творчески использовать саму сцену боя как оружие. Игрок может написать «я опрокидываю горящую жаровню на гоблинов» — ИИ назначит проверку Атлетики чтобы толкнуть тяжёлую железную чашу, и при успехе на области перед персонажем возникнет эффект огня на несколько клеток, а гоблины получат урон и возможно загорятся. «Я обрубаю верёвку держащую люстру над головой орка» — проверка Ловкости или Атака чтобы перебить верёвку, и тяжёлая железная люстра падает на орка нанося дробящий урон. «Я переворачиваю тяжёлый дубовый стол и прячусь за ним от стрел» — стол становится укрытием дающим бонус к AC от дальних атак пока персонаж находится за ним. «Я сталкиваю врага в пропасть» — проверка Атлетики против Атлетики или Акробатики врага, и при успехе враг летит вниз получая урон от падения. --- ЧАСТЬ 19. АФК И ЗАМЕНА ИГРОКА Система обнаружения отсутствия игрока работает на двух таймерах. В режиме исследования, когда партия путешествует или осматривает локацию, игрок считается отошедшим от клавиатуры если не совершал никаких действий в течение двух минут подряд. В режиме боя где каждый ход на счету таймаут сокращается до тридцати секунд. Когда игрок достигает порога, сервер помечает его как АФК: на нижней панели иконка персонажа становится серой и над ней появляется маленькая иконка песочных часов. Персонаж продолжает стоять на месте; если в этот момент идёт бой, персонаж автоматически пропускает свои ходы и не совершает действий. Любой игрок может уйти добровольно нажав кнопку «АФК» в меню настроек. В отличие от автоматического обнаружения, добровольный уход передаёт персонажа под управление ИИ немедленно без периода ожидания. ИИ анализирует последние двадцать действий которые совершил игрок чтобы понять его стиль игры. Если игрок всегда бросался в гущу боя первым, агрессивно атаковал самыми сильными способностями и никогда не использовал защитные действия — ИИ продолжит играть агрессивно. Если игрок предпочитал держаться на расстоянии, использовал лук или заклинания, прятался за укрытиями — ИИ продолжит осторожную тактику. ИИ не будет тратить дорогие одноразовые предметы (зелья редкого лечения, свитки с мощными заклинаниями) и расходовать ячейки заклинаний высших кругов без крайней необходимости — только если жизни персонажа угрожает реальная опасность. При обрыве соединения — игрок вылетел из-за проблем с интернетом или краша клиента — у него есть ровно две минуты чтобы переподключиться. Сервер отсчитывает эти две минуты; персонаж в это время стоит на месте и не совершает действий (не атакует, не убегает, просто замирает). Если в течение двух минут игрок возвращается в игру, он продолжает с того же момента без каких-либо потерь. Если две минуты истекли а игрок не вернулся, начинается голосование среди оставшихся активных игроков. Каждый активный игрок получает в чате сообщение: «Игрок [имя] отсутствует более 2 минут. Заменить его персонажа на ИИ? Да / Нет». Для замены требуется согласие семидесяти пяти процентов голосующих: трое из четырёх, или двое из двух если в партии изначально было двое и один вылетел. Если активных игроков осталось меньше двух, голосование не начинается и персонаж просто ждёт возвращения игрока. Замена игрока до начала партии возможна ещё в главном меню. Если назначенное время начала кампании пришло, а один или несколько игроков не появились, ведущий или любой присутствующий игрок может инициировать голосование. Если семьдесят пять процентов присутствующих согласны начать без опоздавших, отсутствующие сразу заменяются на ИИ-персонажей которые присоединятся к партии с самого начала приключения. --- ЧАСТЬ 20. БАНЫ, ЛИМИТЫ, АНТИЧИТ Система защиты от злоупотреблений построена на трёх уровнях. Первый уровень ограничивает частоту действий: каждый игрок может отправить не более ста двадцати действий в час через Чат №1. Это примерно два действия в минуту, что с запасом покрывает потребности самой активной игры но предотвращает спам-атаки и попытки завалить сервер тысячами запросов. При первом превышении лимита игрок получает предупреждение в чат. При втором превышении в течение того же часа наступает временный бан на тридцать минут — персонаж замирает и не может действовать, но остаётся в игре. При систематических нарушениях (три и более случаев за неделю) игрок получает перманентный бан с невозможностью создать нового персонажа. Второй уровень это дневной лимит игрового времени. Каждый игрок может находиться в игре не более двух часов в сутки. Это ограничение введено для справедливого распределения ограниченного ресурса бесплатных API среди всех игроков. Лимит отслеживается на сервере и сбрасывается в полночь по времени сервера. За пятнадцать минут до исчерпания лимита игрок получает мягкое уведомление в чат: «До окончания игровой сессии осталось 15 минут». За пять минут предупреждение становится жёлтым и настойчивым, и сервер автоматически инициирует сцену костра — все персонажи телепортируются к безопасному месту для финального саммари. При полном исчерпании лимита игра сохраняется и игрок перенаправляется в главное меню. Третий уровень это античит который проверяет что игроки не пытаются обойти игровые механики техническими средствами. Сервер отслеживает скорость ввода: новое действие не может быть отправлено быстрее чем через десять секунд после предыдущего без паузы — это предотвращает макросы и ботов. Сервер детектирует повторяющиеся идентичные действия которые могут указывать на автоматизацию. Сервер валидирует все DDS-команды поступающие от ИИ перед тем как их исполнить, чтобы убедиться что злоумышленник не вставил в ответ ИИ вредоносную команду. --- ЧАСТЬ 21. РЕЖИМ НОВИЧКА Режим новичка это встроенная система обучения для людей которые никогда раньше не играли в Dungeons & Dragons и не знакомы с концепцией настольных ролевых игр. Он автоматически активируется при первом запуске игры и последовательно объясняет все фундаментальные концепции. Обучение начинается с самого базового объяснения: «Dungeons & Dragons это настольная ролевая игра. Представьте что вы собрались с друзьями за столом, и один из вас — ведущий — описывает мир и ситуации. Вы говорите ведущему что делает ваш персонаж, и ведущий рассказывает что происходит в ответ. В нашей игре роль ведущего выполняет искусственный интеллект. Вы пишете свои действия в чат, и ИИ отвечает вам описывая результат.» Это снимает главный барьер непонимания у новых игроков которые привыкли к компьютерным играм с кнопками и джойстиками. Затем объясняется система бросков: «Главный элемент механики D&D — это двадцатигранный кубик, d20. Когда ваш персонаж пытается сделать что-то с неопределённым исходом — ударить врага мечом, взломать замок отмычками, убедить стражника пропустить вас — бросается d20. Чем выше выпавшее число, тем лучше результат. К броску прибавляются модификаторы от характеристик вашего персонажа. Если итоговая сумма равна или выше числа сложности, действие удалось. Если ниже — провалилось. Натуральная двадцатка (выпало ровно 20) — всегда успех, часто с дополнительным положительным эффектом. Натуральная единица (выпало ровно 1) — всегда провал, даже если ваши модификаторы высоки.» Затем объясняются шесть базовых характеристик: «Сила определяет физическую мощь вашего персонажа. Чем выше Сила, тем сильнее вы бьёте в ближнем бою, тем больше можете унести, тем выше прыгаете. Ловкость это проворство и координация — она помогает уклоняться от ударов, метко стрелять из лука, красться в тени, и быстро реагировать в бою. Телосложение это выносливость и здоровье — чем оно выше, тем больше у вас хитов, и тем труднее вас отравить или оглушить. Интеллект это ум и память — он нужен волшебникам для заклинаний, и помогает знать историю, магию, природу. Мудрость это интуиция и сила воли — она нужна жрецам и друидам, помогает замечать скрытое и сопротивляться магическому контролю. Харизма это сила личности — убеждение, обман, запугивание, и заклинания колдунов и бардов.» Затем игроку предлагается создать первого персонажа с пошаговыми подсказками. После создания персонажа начинается короткая обучающая сцена где ИИ в безопасной обстановке показывает как работает игра: «Ты стоишь перед закрытой дверью. Что ты делаешь?» Игрок пробует написать действие, ИИ обрабатывает его и объясняет что произошло и почему. Затем предлагается простой бой с тренировочным манекеном чтобы освоить боевую систему. После завершения обучения игрок может либо продолжить в полноценную кампанию, либо выйти в главное меню. Режим новичка даёт несколько практических преимуществ которые отключаются после трёх проведённых игровых сессий: таймер хода в бою увеличен с тридцати до шестидесяти секунд чтобы было время прочитать подсказки и подумать; над персонажем отображается зелёный значок новичка видимый другим игрокам; при первой встрече с каждой новой механикой появляется краткое пояснение. Режим можно отключить досрочно в настройках. --- ЧАСТЬ 22. ПОВЫШЕНИЕ УРОВНЯ Персонажи получают опыт за победу над врагами (количество опыта зависит от сложности противника), за выполнение квестов, за решение загадок и головоломок, и за особые достижения. Когда накопленный опыт достигает определённого порога, персонаж повышает уровень. Таблица опыта для D&D 5.0: 1 уровень — 0 опыта (стартовый), 2 уровень — 300 XP, 3 уровень — 900 XP, 4 уровень — 2 700 XP, 5 уровень — 6 500 XP, 6 уровень — 14 000 XP, 7 уровень — 23 000 XP, 8 уровень — 34 000 XP, 9 уровень — 48 000 XP, 10 уровень — 64 000 XP, 11 уровень — 85 000 XP, 12 уровень — 100 000 XP, 13 уровень — 120 000 XP, 14 уровень — 140 000 XP, 15 уровень — 165 000 XP, 16 уровень — 195 000 XP, 17 уровень — 225 000 XP, 18 уровень — 265 000 XP, 19 уровень — 305 000 XP, 20 уровень — 355 000 XP. На каждом уровне персонаж получает определённые улучшения в зависимости от своего класса. Первый уровень даёт стартовые способности класса и расы — то с чем персонаж начинает свой путь. Второй уровень открывает первые классовые улучшения: у воина это Второе дыхание позволяющее восстановить немного здоровья прямо в бою, у плута это Хитрое действие позволяющее совершать дополнительные манёвры. Третий уровень это ключевой выбор архетипа (подкласса) который определяет специализацию персонажа на всю оставшуюся игру — воин выбирает между Чемпионом (увеличенный шанс критического удара), Мастером Боя (специальные боевые манёвры), и Мистическим Рыцарем (сочетание боя и магии). Четвёртый, восьмой, двенадцатый, шестнадцатый и девятнадцатый уровни дают повышение характеристик: можно добавить два пункта к одной характеристике или по одному пункту к двум разным, или вместо повышения взять особую черту (feat). Пятый, одиннадцатый и семнадцатый уровни дают воинским классам дополнительные атаки, а заклинателям — доступ к заклинаниям следующих кругов. Бонус мастерства повышается на пятом, девятом, тринадцатом и семнадцатом уровнях. Игрок может заранее спланировать развитие своего персонажа через меню «План прокачки» в профиле. В этом меню для каждого будущего уровня можно указать что именно будет выбрано: какую характеристику повысить, какую черту взять, какой архетип выбрать, какие заклинания выучить. Когда уровень действительно повышается, сервер автоматически применяет запланированные настройки не спрашивая игрока — это ускоряет процесс и позволяет не прерывать игру на долгие раздумья. МАСТЕРСКАЯ КОНТЕНТА Мастерская это специальное пространство куда игрок попадает из главного меню, нажав кнопку «Мастерская». При входе в мастерскую игрок переносится в отдельную сцену оформленную как комната ремесленника — с большим деревянным верстаком в центре, полками с инструментами вдоль стен, магической лампой под потолком которая даёт ровный яркий свет, и большим окном через которое видно как проплывают облака. В этой комнате игрок может свободно двигать блоки как в строительном редакторе — брать блоки из палитры, ставить их на сцену, вращать, менять размер, цвет и тип. Можно размещать готовые шаблоны из библиотеки (таверны, храмы, деревья, монстров, предметы). Всё что игрок строит в мастерской, он может сохранить как шаблон и позже использовать в своих кампаниях или опубликовать в общественную мастерскую. Рядом с верстаком находится интерфейс общения с ИИ-консультантом через чат. ИИ в мастерской это не тот же ИИ который ведёт кампанию — у него отдельный промпт заточенный на помощь в творчестве. Игрок может написать: «Придумай мне босса-некроманта для подземелья пятого уровня, со способностью призывать скелетов и аурой страха», и ИИ немедленно сгенерирует полные характеристики: все шесть статов, хиты, класс брони, две-три уникальные атаки, особые способности с механикой, сопротивления, уязвимости, и даже текстовое описание внешности для нарратива. Или игрок может написать: «Создай легендарный меч который питается душами и даёт владельцу способность видеть в темноте и раз в день телепортироваться на тридцать футов», и ИИ сгенерирует предмет с полными характеристиками, требованиями и балансом соответствующим легендарной редкости. Из личной мастерской можно перейти в общественную мастерскую — это библиотека всего контента созданного всеми игроками и ИИ. Общественная мастерская это отдельный интерфейс где контент представлен в виде карточек с названием, автором, рейтингом и кратким описанием. Доступна сортировка по популярности, по дате добавления, по рейтингу. Работает система фильтров: «Всё» показывает весь контент без ограничений; «Нейросеть» показывает только то что сгенерировал ИИ по запросам игроков в мастерской; «Хоумбрю» показывает только то что создали сами игроки вручную; «Сбалансирован» показывает контент прошедший проверку сервером и признанный соответствующим официальному балансу D&D; «Тест» показывает контент ожидающий проверки. Каждый игрок может опубликовать не более одного предмета в час чтобы предотвратить замусоривание библиотеки низкокачественными набросками. ИИ не имеет такого ограничения и может генерировать контент по запросам без лимитов. Все Homebrew-творения проходят автоматическую проверку баланса на сервере. Сервер сравнивает числовые параметры созданного существа или предмета с эталонными значениями из официального бестиария и списка артефактов соответствующего уровня. Проверяются ключевые метрики: средний урон за раунд, класс брони, количество хитов, бонус атаки, DC способностей. Если все параметры укладываются в допустимый диапазон (плюс-минус десять процентов от эталона), контент получает пометку «Сбалансирован». Если какая-то метрика выходит за границы диапазона, контент остаётся с пометкой «Тест» и предупреждением для пользователей. Если игроки используют несбалансированный контент в своей кампании, и при этом контент радикально ломает игру (например оружие с уроном как у заклинания девятого круга но доступное с первого уровня), вся кампания получает статус «Сломано». С этим статусом награды не засчитываются, кампания исключается из рейтингов, персонажи не могут участвовать в топах. В мастерской также доступен режим симуляции битв. Игрок может создать арену — небольшую сцену из блоков и шаблонов — разместить на ней своего персонажа (или нескольких персонажей) и противников, и запустить автоматический бой. ИИ управляет обеими сторонами: принимает решения за всех участников, делает броски через сервер, описывает ход сражения. Игрок наблюдает за битвой как зритель, может ставить на паузу, перематывать назад, менять условия и перезапускать. НАСТРОЙКИ ИГРЫ Меню настроек открывается нажатием на кнопку с иконкой шестерёнки которая находится в верхней части правой панели. Если партия спасла деревню Хиллтоп от гоблинов — флаг «village_hilltop_saved» устанавливается в true, и при следующем посещении игроки увидят что деревня восстановлена, жители построили статую героям на центральной площади, а в таверне их будут кормить бесплатно. Если партия обокрала храм Света — флаг «temple_robbed» становится true, стража во всех городах получает ориентировки на святотатцев, жрецы отказывают в услугах. Если партия убила лидера гильдии воров — флаг «thief_guild_leader_slain» становится true, гильдия объявляет награду за головы убийц, на партию могут напасть наёмные убийцы в любой момент. --- ЧАСТЬ 27. РОЛЕВОЙ ОТЫГРЫШ Вдохновение это особая награда за качественный отыгрыш персонажа. Ведущий или другой игрок (один раз за игровую сессию) может выдать вдохновение если персонаж игрока действует в соответствии со своим характером, идеалами, привязанностями или недостатками даже когда это неоптимально с точки зрения игровой механики. Например персонаж-трус отказывается заходить в тёмную пещеру один, хотя все остальные уже внутри и ждут его — это отыгрыш недостатка, который заслуживает вдохновения. Персонаж жертвует своей атакой чтобы вытащить раненого товарища из-под огня — это отыгрыш привязанности (верность друзьям). Персонаж отказывается от взятки потому что верит в справедливость — это отыгрыш идеала. Вдохновение позволяет однократно перебросить любой бросок двадцатигранного кубика. Игрок делает бросок, видит результат, и если результат ему не нравится — он может объявить что тратит вдохновение и бросить кубик ещё раз, выбрав какой из двух результатов использовать. Нельзя потратить вдохновение после того как ведущий объявил исход броска — решение нужно принимать сразу. Визуально вдохновение отображается на правой панели как маленький золотой горящий кубик рядом с именем персонажа. Когда игрок тратит вдохновение, над его персонажем на 3D-сцене появляется призрачный двадцатигранник который вспыхивает золотым светом и рассыпается искрами. Эта анимация видна всем игрокам в партии и служит сигналом «я только что использовал вдохновение». Недостатки персонажа это негативные черты характера которые игрок выбирает при создании персонажа. Они делают персонажа живым и интересным, создают почву для драматических моментов и сложных решений. Недостатки могут быть разными: Жадность — персонаж испытывает почти непреодолимое желание забрать всё ценное что видит, даже если это принадлежит другим, даже если это очевидно плохая идея (например снять дорогое кольцо с пальца статуи которая может быть гаргульей). Вспыльчивость — персонаж реагирует на любое оскорбление или провокацию немедленной агрессией, часто физической, даже когда драка невыгодна или опасна. Алкогольная зависимость — персонаж ищет выпивку в каждом посещённом городе, может напиться в неподходящий момент, тратит деньги на алкоголь в ущерб снаряжению. Фобия — персонаж панически боится чего-то конкретного: высоты, пауков, замкнутых пространств, воды, темноты. При столкновении с объектом фобии персонаж должен пройти спасбросок Мудрости чтобы сохранить контроль над собой. Клятва — персонаж дал нерушимую клятву которая ограничивает его действия: никогда не отступать, никогда не лгать, всегда помогать нуждающимся, никогда не использовать яды, никогда не носить оружие. ИИ напоминает о недостатках не чаще одного раза за игровую сессию чтобы они не становились раздражающими. Напоминание происходит в драматически подходящий момент когда недостаток органично вписывается в сцену — не «кстати, твой персонаж жадный, отдай мне все деньги», а «ты видишь роскошное золотое ожерелье на шее спящего дракона, и твои пальцы сами тянутся к нему». Игрок может попытаться преодолеть недостаток — сделать бросок Мудрости против DC 12 чтобы подавить impulse — или поддаться недостатку и автоматически получить вдохновение за хороший отыгрыш. Сцена у костра это ритуал завершения игровой сессии. Когда игровой лимит подходит к концу или партия решает сделать привал на ночлег, все персонажи автоматически собираются вокруг костра. Визуально сцена меняется: если персонажи были в подземелье, игра показывает что они нашли безопасную пещеру и развели огонь; если в лесу — поляна с большим костром; если в городе — комната в таверне с камином. Персонажи рассаживаются вокруг огня, им предлагается еда и питьё (автоматически, не расходуя ресурсы инвентаря). ИИ-ведущий генерирует краткое саммари прошедшей сессии: какие ключевые события произошли, кто из персонажей особенно отличился, какие враги были повержены, какие загадки остались неразгаданными, какие решения могут иметь последствия в будущем. Например: «Вы вошли в Гоблинскую пещеру, сразились с патрулём гоблинов и их вожаком, освободили пленного кузнеца из деревни, и нашли странный амулет покрытый непонятными рунами. Кузнец рассказал что в глубине пещеры есть ещё что-то — что-то что даже гоблины боятся потревожить. Вы решили отложить исследование на завтра и вернуться в деревню чтобы подготовиться.» После саммари каждому игроку даётся слово. Игрок может написать в чат что его персонаж говорит или делает у костра: делится мыслями о прошедшем дне, обсуждает планы на будущее, разговаривает с товарищами, задаёт вопросы NPC если они присутствуют у костра. Это момент для спокойного ролевого отыгрыша без боя и спешки. Сразу после сцены у костра показывается сцена после титров. Экран плавно затемняется до полностью чёрного, затем медленно проявляется короткая сцена — не длиннее пятнадцати-двадцати секунд. Эта сцена показывает что произойдёт в ближайшем будущем игрового мира, в следующей сессии. Злодей в своём логове получает доклад от шпиона: «Герои убили вожака гоблинов. Они нашли амулет. Они ещё не знают что он делает, но скоро узнают. Послать за ними охотников?» Тёмная фигура на троне отвечает: «Нет. Пусть думают что они в безопасности. Амулет приведёт их прямо ко мне.» Или союзный NPC находит важную улику: «Я изучил руны на амулете который вы принесли. Это ключ к Гробнице Короля-лича. Если активировать его в полнолуние...» Или в покинутой игроками локации происходит некое событие: проход который они не исследовали начинает светиться, и из него доносится низкий гул. Сцена обрывается на самом интригующем моменте, и появляется окно о завершении игрового лимита. Сны во время отдыха могут содержать сюжетные подсказки. Когда персонаж спит, ИИ делает скрытый бросок d20. При результате 15 или выше (для обычных персонажей) или 10 или выше (для жрецов, паладинов, друидов и колдунов которые ближе к божественному и мистическому), персонаж видит сон. Обычные сны просто описывают странные, сюрреалистические образы которые создают атмосферу. Вещие сны содержат символические намёки на будущие события — метафоры и аллегории которые невозможно расшифровать сразу но которые обретают смысл позже. Сон показывается в Чате №1 когда персонаж просыпается: «Ты просыпаешься с ощущением что видел что-то важное. Во сне ты стоял на берегу чёрного озера, и из глубины поднимались пузыри. Огромный глаз открылся под водой и посмотрел на тебя. Затем ты проснулся.» --- ЧАСТЬ 28. ХАРДКОР И НАСЛЕДИЕ Режим «Железный герой» создан для игроков которым недостаточно обычной сложности и которые хотят настоящего напряжения, страха потери и реализма. В этом режиме смерть персонажа окончательна и необратима никакими средствами. Магия воскрешения не работает — ни Revivify, ни Raise Dead, ни Resurrection, ни даже Wish. Нельзя загрузить сохранение и переиграть. Каждое решение может действительно стать последним. В награду за этот постоянный риск персонаж получает на двадцать пять процентов больше опыта за все действия, что позволяет быстрее развиваться и компенсировать повышенную опасность. В режиме «Железный герой» действует детализированная система травм которая делает каждое ранение потенциально опасным. Обычный режим отделывается шрамами и хромотой — это чисто нарративные эффекты которые не влияют на механику, просто описаны словами. В хардкорном режиме травмы имеют реальные механические последствия с которыми приходится жить и играть. Критическое попадание от дробящего оружия (молот, булава, падение камня на голову, удар щитом) ломает кости. Перелом руки: скорость атаки оружием в этой руке снижается на 25 процентов, использование двуручного оружия становится невозможным (нужны обе руки а одна сломана), проверки Атлетики и Акробатики требующие рук проходят с помехой. Перелом ноги: скорость передвижения снижается вдвое, проверки Акробатики на удержание равновесия с помехой, нельзя использовать действие Рывок и Отход. Перелом рёбер: при получении урона нужно проходить спасбросок Телосложения DC 12 иначе персонаж теряет действие на следующем ходу от боли. Травмы лечатся долго: месяц внутриигрового покоя в храме или госпитале, или мощная магия вроде Greater Restoration (5-й круг заклинаний, доступно только с 9 уровня). Критическое попадание от рубящего оружия оставляет глубокую кровоточащую рану. Персонаж теряет 1d4 хитов в начале каждого своего хода из-за кровопотери. Чтобы остановить кровь, нужна проверка Медицины DC 12 действием и расход одного бинта или чистой ткани. Если рану не перевязать, кровотечение продолжается пока персонаж не умрёт или не получит магическое лечение. Критическое попадание от огня (заклинание, драконье дыхание, падение в лаву) вызывает серьёзные ожоги. Ожог лица: помеха на все проверки Убеждения и Выступления из-за обезображенного вида. Ожог руки: помеха на все проверки Ловкости рук и использование инструментов. Ожоги также дают уязвимость к следующей атаке огнём пока не заживут (одна неделя покоя или магия). Падение с высоты более 50 футов с проваленным спасброском Телосложения (DC 15) может сломать позвоночник. Персонаж парализован ниже пояса и не может ходить, только ползти со скоростью 5 футов. Это требует либо мощной магии (Regeneration, 7-й круг), либо месяца внутриигрового покоя в храме с постоянным уходом. Система наследия создана чтобы игрок не вылетал из кампании после смерти персонажа а мог продолжить игру. Когда персонаж погибает в режиме «Железный герой» или в любой кампании если игрок этого хочет, можно создать наследника. Наследник это новый персонаж который связан с погибшим предшественником: его сын или дочь, младший брат или сестра, ученик, друг детства, или просто человек которому погибший однажды спас жизнь и который решил продолжить его дело. Наследник получает на один уровень выше чем был у погибшего персонажа на момент смерти. Это не мгновенный скачок силы а отражение того что наследник не новичок — он уже прошёл свой путь и достиг определённого мастерства, просто теперь он присоединяется к этому конкретному приключению. Наследник получает один артефакт предшественника (по выбору игрока) — семейную реликвию которая передаётся из поколения в поколение. Наследник получает личный квест: отомстить убийце предшественника (если убийца известен), завершить незаконченное дело (если предшественник погиб не завершив важный квест), найти и вернуть утерянный артефакт, или восстановить честь семьи (если предшественник погиб с позором). Эпилог персонажа генерируется ИИ когда кампания завершается или игрок навсегда покидает игру. Это текст из пяти-десяти строк описывающий дальнейшую судьбу персонажа после того как активные приключения закончились. Эпилог основывается на всех решениях которые игрок принимал во время игры: его действиях, выборах, отношениях с NPC, выполненных и проваленных квестах. Воин который всегда защищал слабых мог стать капитаном городской стражи, жениться на спасённой принцессе, и прожить долгую счастливую жизнь. Воин который был жесток и жаден мог спиться в дешёвой таверне и умереть в нищете, вспоминая былую славу и проклиная тот день когда продал душу за золото. Волшебник который посвятил жизнь поиску знаний мог основать академию магии и обучить новое поколение магов. Волшебник который искал бессмертия любой ценой мог исчезнуть в попытке стать личем — и никто не знает удалась ли попытка. Плут который воровал только у богатых и помогал бедным мог стать народным героем, легендарным Робин Гудом своего мира. Плут который воровал у всех подряд мог кончить жизнь на виселице. Эпилог сохраняется в профиле персонажа навсегда. --- ЧАСТЬ 29. РЕСУРСЫ И ВЫЖИВАНИЕ Учёт расходуемых ресурсов создаёт дополнительный слой геймплея связанный с планированием и выживанием. Каждый игровой день персонаж должен потребить один паёк еды и один бурдюк воды (примерно галлон). Если персонаж не получает достаточно еды, он начинает страдать от голода: после первого дня без еды — один уровень истощения (помеха на проверки), после нескольких дней — дополнительные уровни вплоть до смерти на шестом уровне истощения. Недостаток воды наступает быстрее и имеет те же эффекты. Еду можно купить в городах и деревнях, добыть охотой (проверка Выживания в лесу), найти в дикой природе (проверка Природы чтобы отличить съедобные ягоды от ядовитых), или получить от NPC в качестве награды. Факелы это ограниченный источник света. Один стандартный факел горит один час реального времени. В инвентаре персонажа показывается точное количество оставшихся факелов. В подземелье без факелов персонажи без ночного видения (люди, дварфы, халфлинги) полностью слепнут и получают помеху на все действия и атаки, а враги получают преимущество на атаки по ним. Персонажи с ночным видением (эльфы, гномы, тифлинги, полуорки) видят в темноте в радиусе шестидесяти футов но только в оттенках серого. Магические источники света (заклинание Light, магический кристалл) не расходуются но занимают руку или требуют концентрации. Верёвки изнашиваются от использования. Каждый раз когда персонажи используют верёвку для чего-то рискованного (спуск в пропасть с нагрузкой, подъём тяжёлого груза, связывание крупного существа), сервер делает скрытый бросок d20. При результате 1-3 верёвка рвётся в самый неподходящий момент. Старая потрёпанная верёвка имеет больший шанс разрыва. Стрелы и болты для арбалета это расходуемые боеприпасы. После боя игрок может попытаться собрать выпущенные стрелы — бросок процента. На открытой местности шанс найти стрелу выше (70%), в густом лесу или подземелье ниже (50%), в воде или лаве стрелы теряются полностью. ИИ отслеживает количество стрел и предупреждает когда запас подходит к концу. Магические компоненты с указанной стоимостью в описании заклинания должны быть куплены заранее и храниться в инвентаре. Жемчужина для Identify (100 золотых), алмаз для Revivify (300 золотых), дорогие масла и благовония для других заклинаний. При использовании заклинания компонент расходуется если это указано в описании. ИИ проверяет наличие компонента в инвентаре и если его нет — заклинание не может быть применено. ИИ в описаниях периодически напоминает о состоянии ресурсов: «Твои факелы почти догорели. В рюкзаке остался всего один. Его хватит на час, может меньше. Возможно стоит зайти в лавку перед тем как спускаться в следующее подземелье.» Это не просто информационное сообщение — это элемент напряжения и планирования. --- ЧАСТЬ 30. Гильдия Торговцев контролирует цены и доступ к товарам во всех крупных городах мира. Это не злая корпорация и не благотворительная организация — это прагматичные бизнесмены которые ценят стабильность, прибыль и репутацию. При отношении 0-20 гильдия отказывается обслуживать партию — ни один уважающий себя торговец не будет рисковать гневом гильдии ради нескольких монет. При отношении 21-40 цены завышены на 25 процентов, ассортимент ограничен только самыми базовыми товарами, торговцы грубы. При 41-60 стандартные цены и стандартный ассортимент. При 61-80 скидка 10 процентов, доступ к редким товарам, торговцы приветливы и могут поделиться рыночными слухами. При 81-100 скидка 25 процентов, доступ к уникальным товарам которые не продаются обычным покупателям, возможность заказать доставку редких предметов из других городов, торговцы первыми предлагают выгодные сделки. Храм Света поклоняется богу света, исцеления и защиты. Жрецы Храма дают обет помогать всем нуждающимся независимо от их веры или происхождения — но у них есть пределы. При отношении 0-20 жрецы отказывают в магическом лечении (но не в обычной медицинской помощи — это нарушило бы их обет). При 21-40 базовое лечение доступно но за полную цену, продвинутые услуги недоступны. При 41-60 стандартные цены на лечение, базовые благословения. При 61-80 бесплатное лечение до определённого уровня, бесплатные благословения перед выходом, возможность попросить жреца сопровождать партию в опасном квесте. При 81-100 полное бесплатное лечение и воскрешение павших, мощные благословения с длительным эффектом, доступ к священным артефактам храма, возможность призвать небесного союзника в критический момент. Академия Магов это башня в столице где архимаги изучают тайны мироздания. В отличие от Храма, Академия не помогает бесплатно — знание это сила, и сила имеет цену. При 0-20 вход в Академию запрещён, стража выпроваживает непрошеных гостей. При 21-40 доступна только публичная библиотека с базовыми книгами. При 41-60 стандартный доступ, можно покупать свитки и зелья. При 61-80 доступ в закрытые секции библиотеки с редкими заклинаниями, возможность попросить архимага идентифицировать сложный магический предмет. При 81-100 доступ к запретной секции где хранятся знания которые не должны покидать стены Академии, возможность изучить уникальные заклинания которых нет в свободном доступе, маги Академии могут телепортировать партию в любую точку континента. Гильдия Воров контролирует теневую экономику — краденое, контрабанда, запрещённые товары, информация которую не найти легально. При 0-20 гильдия объявляет партию врагами — наёмные убийцы могут атаковать в любой момент, воры срезают кошельки и портят снаряжение, информаторы дают ложные наводки. При 21-40 воры держатся подальше, услуги недоступны. При 41-60 базовый доступ к теневым рынкам, стандартные цены на информацию и запрещённые товары. При 61-80 скидки на краденое, доступ к более серьёзной информации, возможность заказать кражу или устранение конкурента. При 81-100 гильдия предоставляет убежище в любом городе где есть их представительство, даёт доступ к сети осведомителей которая может найти кого угодно и где угодно, лучшие воры могут сопровождать партию во взломе или скрытном проникновении. Отношения с фракциями меняются в зависимости от действий. Помощь фракции (выполнение квеста, пожертвование, спасение члена фракции) повышает отношение на 15-25 пунктов в зависимости от значимости помощи. Вред фракции (кража, убийство члена, работа на конкурентов) снижает отношение на 20-30 пунктов. Нейтральные действия не влияют на отношение. Фракции также общаются между собой: если партия помогла Храму Света, Гильдия Торговцев может узнать об этом и немного повысить отношение (герои с хорошей репутацией — хорошие клиенты). Если партия ограбила Академию Магов, Храм Света может снизить отношение (они не одобряют кражу знаний). --- ЧАСТЬ 31. ПРОРОЧЕСТВО При создании кампании ведущий или первый игрок может выбрать опцию «Пророчество: включено». Если эта опция выбрана, ИИ генерирует в самом начале кампании таинственное предсказание — короткий текст из четырёх-шести поэтических строк, написанный туманным, многозначным языком. Пророчество нельзя понять полностью сразу — его метафоры и образы можно будет расшифровать только по мере того как сюжет кампании развивается и предсказанные события начинают сбываться на глазах у игроков. Пророчество генерируется ИИ на основе случайных элементов и всегда уникально для каждой кампании. Оно записывается в World State и передаётся ИИ в контексте каждого запроса. ИИ хранит пророчество в памяти и периодически напоминает о нём в ключевые моменты — когда происходящее в игре начинает совпадать с предсказанным. «Строки древнего пророчества внезапно всплывают в твоей памяти: "кровь невинных откроет врата, и Спящий проснётся". Ты смотришь на лужу крови растекающуюся от тела жертвенного агнца на алтаре, и видишь как каменная плита под алтарём начинает светиться.» Пророчество не является жёстким сценарием который обязательно сбудется. Это скорее намёк на возможное будущее, которое может измениться в зависимости от действий игроков. ИИ может адаптировать значение пророчества под то что реально происходит в кампании. --- ЧАСТЬ 32. ЖУРНАЛЫ И ДНЕВНИКИ Общий дневник партии ведётся автоматически сервером без участия игроков. Каждое значимое событие записывается в хронологическом порядке: победа над боссом (каким именно, где, какой был состав партии, кто нанёс решающий удар), нахождение важного артефакта (что за предмет, где найден, какие свойства известны), ключевое сюжетное решение (помочь фракции А или фракции Б, пощадить врага или казнить), смерть персонажа если она произошла, получение нового квеста. Дневник структурирован по игровым дням. Каждый день имеет заголовок — например «День 7: Гоблинская пещера» или «День 23: Аудиенция у короля». Под заголовком перечислены события дня. Игроки могут открыть дневник в любой момент через меню и пролистать записи чтобы освежить в памяти что произошло несколько сессий назад — это особенно полезно для долгих кампаний которые длятся месяцы реального времени. Дневник нельзя редактировать — это объективная хроника произошедшего, а не субъективный дневник персонажа. Для личных заметок игроки могут использовать Чат №3 или внешние инструменты. --- ЧАСТЬ 33. ГЕНЕРАТОР ИМЁН Генератор имён работает по простой но эффективной формуле: прилагательное плюс существительное. Эта схема веками используется в фэнтезийной литературе и играх потому что она создаёт запоминающиеся, образные названия которые сразу вызывают в голове картинку. «Гремящий Холм» — возвышенность куда часто бьют молнии, или место где когда-то произошла великая битва и земля до сих пор дрожит. «Чёрный Клык» — таверна с дурной репутацией в портовом квартале, или оружие злого воина, или название банды орков. «Плачущая Вдова» — таверна где подают лучшее тёмное пиво, или статуя в храме оплакивающая павших героев, или заброшенный особняк с трагической историей. «Золотой Чертог» — дворец короля, или главный зал дворфийской крепости, или название банка Гильдии Торговцев где хранят сокровища. «Сломанная Стрела» — таверна у городских ворот, или перевал в горах где лучники короля потерпели поражение от орков, или название отряда наёмников потерявших командира. Генератор используется для названий: городов и деревушек, таверн и постоялых дворов, значимых NPC (вместо личных имён для загадочных персонажей — «Человек в Чёрном», «Странник с Севера»), легендарного оружия и артефактов, природных объектов (гор, лесов, рек, перевалов), и даже целых регионов. Имена персонажей игроки выбирают сами или вводят вручную. Генератор предлагает варианты имён подходящие под выбранную расу: для людей это обычные фэнтезийные имена (Алан, Эдвин, Маркус, Катерина, Элиза), для эльфов мелодичные с апострофами (Эль'андар, Лили'эт, Тар'эниэль), для дварфов короткие и грубые (Торин, Брун, Грета, Дагна), для драконорождённых составные (Красный Коготь, Золотая Чешуя, Чёрный Клык), для тифлингов инфернальные (Зара, Мордекай, Азариэль, Лилит). --- ЧАСТЬ 34. СЦЕНА ПОСЛЕ ТИТРОВ Сцена после титров это короткая загадочная сцена которая показывается игрокам ПОСЛЕ сцены у костра и ДО появления финального окна о завершении игровой сессии. Она служит мостиком между текущей сессией и следующей — создаёт интригу, напоминает о неразрешённых сюжетных линиях, подогревает интерес. Технический процесс генерации: когда таймер отсчитывает последние пять минут до конца лимита, сервер отправляет ИИ отдельный запрос с пометкой [СЦЕНА ПОСЛЕ ТИТРОВ]. В запросе содержится текущий контекст игры и краткое саммари сессии (которое ИИ уже сгенерировал для сцены костра). ИИ получает задание: создать короткую сцену которая покажет что произойдёт в ближайшем будущем. Сервер сохраняет эту сцену и воспроизводит в нужный момент. Сцена показывается так: экран плавно затемняется до полностью чёрного в течение двух секунд. Зловещая или интригующая музыка начинает играть на фоне. Экран медленно проявляется, показывая короткую сцену — диалог, действие, или просто визуальную метафору. Сцена длится пятнадцать-двадцать секунд, обрывается на самом интересном моменте (клиффхэнгер), и экран снова затемняется. Затем появляется окно «Дневной лимит игры исчерпан. Прогресс сохранён. До сброса лимитов осталось X часов YY минут. [Выйти в главное меню]». Содержание сцены всегда связано с тем что происходит в кампании. Злодей получает доклад от своего шпиона — он узнаёт о том что партия приближается к его логову и готовит ловушку. Союзный NPC находит важную информацию которая изменит ход расследования — он спешит сообщить партии но не знает где они сейчас. В покинутой игроками локации происходит событие — потайной проход который они не заметили открывается сам собой, магический барьер который они не смогли преодолеть тускнеет и исчезает. Артефакт который партия носит с собой начинает светиться в темноте — он реагирует на что-то, что приближается. Неизвестный наблюдатель следил за партией всё это время — сцена показывает его глазами как он записывает в книгу: «Субъекты покинули пещеру. Амулет у них. Доложу Хозяину.» --- ЧАСТЬ 35. РИТУАЛЫ ВНЕ БОЯ Некоторые заклинания в правилах D&D имеют специальное свойство «ритуал» (ritual). Это означает что заклинатель может прочитать такое заклинание не тратя ячейку, но это занимает дополнительные десять минут сверх обычного времени применения. Механика ритуалов предназначена для утилитарных заклинаний которые полезны но не критичны в бою: Обнаружение магии (Detect Magic), Понимание языков (Comprehend Languages), Поиск питомца (Find Familiar), Создание воды (Create Water), Обнаружение яда (Detect Poison), Магический рот (Magic Mouth), Тензеров парящий диск (Floating Disk), и другие. Когда игрок хочет использовать заклинание с меткой «ритуал» и при этом партия не находится в бою, сервер показывает диалоговое окно с выбором: «Как вы хотите применить [название заклинания]? [Потратить ячейку — мгновенно] [Прочитать как ритуал — займёт 10 минут, ячейка не тратится] [Отмена]». Если игрок выбирает ритуал, игровое время перематывается на десять минут вперёд — сервер проверяет случайные встречи в зависимости от опасности локации, обновляет состояния NPC, и применяет заклинание. ИИ может активно предлагать использовать ритуал когда это уместно: «Вы стоите перед запечатанной дверью покрытой странными рунами. Если хотите, вы можете прочитать Обнаружение Магии как ритуал чтобы понять какая именно магия на двери, не тратя ячейку. Это займёт десять минут — но вы в безопасности и можете себе это позволить.» --- ЧАСТЬ 36. ПЕСОЧНИЦА Режим песочницы открывается из главного меню выбором соответствующего пункта при создании новой кампании. Перед нажатием кнопки «Начать» показывается описание: «Режим для ведущего без ограничений и правил. Создавайте любые сцены, тестируйте механики, экспериментируйте. Никаких сохранений в кампанию — всё созданное исчезнет при выходе. Время в песочнице учитывается в дневном лимите игры.» В песочнице ведущий получает полный набор инструментов для творчества. Можно создавать любых существ в любых количествах: пятьдесят драконов против одного гоблина чтобы посмотреть кто победит, армия скелетов против отряда паладинов, или точная копия партии игроков против их зеркального отражения. Можно выдавать любые предметы: легендарный меч +3 вызывающий молнии с первого уровня, кольцо делающее носителя невидимым навсегда, зелье превращающее выпившего в дракона. Можно менять погоду и время суток мгновенно: устроить грозу в помещении, сделать ночь посреди дня, вызвать метель в пустыне. Интерфейс песочницы включает панель управления с кнопками. Пауза замораживает всех существ на сцене — они останавливаются в текущих позах и не двигаются пока пауза не будет снята. Ускорение времени позволяет прокрутить длительные процессы: скорость ×2, ×5, ×10, ×100. Покадровый режим продвигает игру на один ход с каждым кликом — полезно для тестирования боевых механик. Свободная камера позволяет летать сквозь стены и наблюдать за сценой с любого ракурса. Строка консоли для прямых команд ИИ и особых команд песочницы. Особые команды песочницы которые обрабатываются сервером напрямую: /clear — полностью очищает сцену от всех объектов и существ; /save имя — сохраняет текущую сцену локально в файл; /load имя — загружает сохранённую сцену; /killall — мгновенно убивает всех существ; /reviveall — воскрешает всех мёртвых существ; /spawn код xколичество — создаёт указанных существ в случайных местах сцены (коды из таблицы шаблонов: TJ для гоблина, TK для орка, TR для дракона и так далее); /noclip on — включает режим полёта сквозь блоки; /noclip off — выключает. В песочнице нет постоянных сохранений — всё созданное существует только пока открыт режим. При выходе всё сбрасывается. Время проведённое в песочнице суммируется с общим дневным лимитом игры. --- ЧАСТЬ 37. БЕСКОНЕЧНЫЙ РЕЖИМ Бесконечный режим полностью убирает из игры сюжет, диалоги, квесты и неигровых персонажей, оставляя только чистый геймплей последовательных сражений. Игроки проходят через бесконечную череду процедурно генерируемых локаций, на каждой из которых их ждут враги. Сложность повышается очень плавно и незаметно — нет резких скачков когда после лёгких гоблинов внезапно появляется древний дракон. Таблица сложности по локациям точно откалибрована для постепенного увеличения напряжения. Локации с первой по десятую: гоблины, кобольды, скелеты, зомби, гигантские крысы, мелкие пауки, волки, бандиты-новички. Это простые противники на которых игроки осваивают базовые механики боя. От трёх до пяти обычных врагов на локацию. Локации с одиннадцатой по двадцатую: орки, хобгоблины, крупные пауки, элементали малые, культисты, гаргульи, вурдалаки, доппельгангеры. Враги начинают использовать особые способности и магию низких кругов. От трёх до шести обычных врагов или один-два элитных. Локации с двадцать первой по тридцатую: минотавры, гидры, вампиры-спавны, големы, духи, малые дьяволы, виверны. Каждый бой требует тактического мышления. ГЛАВНОЕ МЕНЮ И МОТИВАЦИЯ Главное меню это первый экран который видит игрок после запуска игры. Это не 3D-сцена а стилизованный интерфейс в духе фэнтези. Фоном служит анимированная карта мира нарисованная в стиле старинных пергаментных карт: с коричневатым оттенком, стилизованными горами, лесами и городами отмеченными значками, с медленно проплывающими облаками и occasionally пролетающим драконом который на секунду появляется в углу карты и исчезает за краем. По карте проложены пунктирные линии обозначающие торговые пути, а в океане нарисованы морские чудовища в стиле средневековых карт. В главном меню расположены следующие элементы: «Начать игру» — большая кнопка в центре экрана, стилизованная под старинную деревянную табличку с золотыми буквами. При нажатии открывает подменю с тремя вариантами. Первый вариант «Новая кампания» открывает интерфейс создания кампании где можно выбрать режим (обычная кампания, песочница, бесконечный режим), редакцию правил (3.5, 5.0 2014, 5.5 2024), придумать название (или оставить поле пустым чтобы ИИ придумал сам), выбрать максимальное количество игроков от одного до десяти, включить или выключить пророчество, выбрать тип доступа (открытая для всех, по коду, только для друзей, по приглашению), и назначить ведущего из числа присутствующих игроков. Второй вариант «Присоединиться по коду» открывает поле ввода шестизначного кода который сгенерировал ведущий другой кампании. Из личной мастерской можно перейти в общественную мастерскую — библиотеку контента созданного всеми игроками. В общественной мастерской контент отображается плиткой с карточками, доступна сортировка и фильтры: «Всё», «Только от ИИ» (сгенерировано нейросетью), «Только Хоумбрю» (создано игроками), «Сбалансирован» (прошло проверку сервером), «Тест» (ожидает проверки).
